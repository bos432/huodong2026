# 服务器重启与日常维护说明

本文档用于说明服务器重启后需要启动和检查哪些服务，以及平时如何维护项目。

当前线上访问地址：

```text
H5：https://rd.chaimen666.com/#/pages/index/index
后台：https://rd.chaimen666.com/admin/
API：https://rd.chaimen666.com/api
```

服务器项目目录：

```text
/www/wwwroot/rd.chaimen666.com
```

## 一、服务器重启后需要启动什么

服务器重启后，主要检查这几个服务：

| 服务 | 作用 | 是否必须 |
| --- | --- | --- |
| Nginx | 负责访问域名、H5、后台、API 反向代理 | 必须 |
| MySQL | 保存后台、小程序、订单、活动等数据 | 必须 |
| Node API | 后端接口服务，监听 `3000` 端口 | 必须 |
| 宝塔面板 | 用于管理网站、数据库、Nginx 配置 | 建议 |

如果 Nginx 和 MySQL 设置了开机自启，服务器重启后通常会自动启动。

最容易漏掉的是：

```text
Node API 后端服务
```

如果 API 没启动，H5 和后台页面可能能打开，但登录、活动列表、报名、上传图片等功能会失败。

## 二、重启后快速检查流程

服务器重启后，按下面顺序检查。

### 1. 进入项目目录

```bash
cd /www/wwwroot/rd.chaimen666.com
```

### 2. 检查 Nginx

```bash
nginx -t
```

如果显示：

```text
syntax is ok
test is successful
```

说明 Nginx 配置正常。

重载 Nginx：

```bash
nginx -s reload
```

如果 Nginx 没启动，可以在宝塔里启动，也可以尝试：

```bash
systemctl start nginx
```

### 3. 检查 MySQL

宝塔里进入：

```text
软件商店 -> MySQL -> 启动
```

命令行可尝试：

```bash
systemctl status mysqld
```

或：

```bash
systemctl status mysql
```

如果没启动：

```bash
systemctl start mysqld
```

或：

```bash
systemctl start mysql
```

不同服务器服务名可能不同，以宝塔面板显示为准。

### 4. 检查 API 是否已启动

查看 3000 端口：

```bash
ss -lntp | grep 3000
```

如果能看到 `3000`，说明 API 大概率已经启动。

也可以直接检查健康接口：

```bash
curl -i http://127.0.0.1:3000/api/health
```

或者：

```bash
curl -i http://127.0.0.1:3000/api/health/ready
```

如果连接失败，说明 API 没启动，需要手动启动。

## 三、启动 API 的方式

推荐使用 PM2 管理 Node API，因为 PM2 可以开机自启、异常自动重启。

### 方式一：PM2 启动，推荐

进入项目目录：

```bash
cd /www/wwwroot/rd.chaimen666.com
```

确认 API 已构建：

```bash
ls -l apps/api/dist/main.js
```

启动 API：

```bash
pm2 start apps/api/dist/main.js --name activity-api
```

查看状态：

```bash
pm2 list
```

查看日志：

```bash
pm2 logs activity-api
```

设置开机自启：

```bash
pm2 startup
pm2 save
```

注意：执行 `pm2 startup` 后，终端可能会输出一行需要复制执行的命令。按它提示执行一次，然后再执行 `pm2 save`。

以后服务器重启后，PM2 会自动拉起 `activity-api`。

### 方式二：PM2 重启已有服务

如果之前已经用 PM2 启动过，重启后只需要：

```bash
pm2 list
pm2 restart activity-api
```

如果服务名不是 `activity-api`，按 `pm2 list` 里显示的名称重启。

### 方式三：临时 nohup 启动

如果服务器没有安装 PM2，可以临时启动：

```bash
cd /www/wwwroot/rd.chaimen666.com
mkdir -p runtime
nohup node apps/api/dist/main.js > runtime/api.log 2>&1 &
```

查看日志：

```bash
tail -f runtime/api.log
```

这个方式可以应急，但不如 PM2 稳定。服务器再次重启后通常需要重新执行。

## 四、启动后必须验证

启动完成后，检查本机 API：

```bash
curl -i http://127.0.0.1:3000/api/health/ready
```

再检查公网访问：

```bash
curl -i https://rd.chaimen666.com/api/health/ready
```

然后浏览器访问：

```text
https://rd.chaimen666.com/#/pages/index/index
https://rd.chaimen666.com/admin/
```

后台能登录，H5 能加载活动列表，说明服务基本正常。

## 五、如果页面能打开但接口失败

这种情况通常是 API 没启动，或者 Nginx `/api/` 代理失败。

按顺序检查：

```bash
ss -lntp | grep 3000
curl -i http://127.0.0.1:3000/api/health
curl -i https://rd.chaimen666.com/api/health
```

如果本机 `127.0.0.1:3000` 不通，启动 API。

如果本机通、域名不通，检查 Nginx 配置里的：

```nginx
location ^~ /api/ {
    proxy_pass http://127.0.0.1:3000/api/;
}
```

修改 Nginx 后执行：

```bash
nginx -t && nginx -s reload
```

## 六、如果后台打不开

确认后台构建文件存在：

```bash
ls -l /www/wwwroot/rd.chaimen666.com/apps/admin/dist/index.html
```

不存在就重新打包：

```bash
cd /www/wwwroot/rd.chaimen666.com
npm --prefix apps/admin run build
```

Nginx 中后台配置应为：

```nginx
location = /admin {
    return 301 /admin/;
}

location ^~ /admin/ {
    alias /www/wwwroot/rd.chaimen666.com/apps/admin/dist/;
    index index.html;
    try_files $uri $uri/ /admin/index.html;
}
```

访问后台时使用：

```text
https://rd.chaimen666.com/admin/
```

注意最后的 `/`。

## 七、如果 H5 首页打不开

确认 H5 构建文件存在：

```bash
ls -l /www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5/index.html
```

不存在就重新打包：

```bash
cd /www/wwwroot/rd.chaimen666.com
npm --prefix apps/mobile run build:h5
```

宝塔网站目录应设置为：

```text
/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5
```

Nginx 根目录应为：

```nginx
root /www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5;
```

并保留：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

## 八、日常维护要做什么

### 每天检查

1. 打开 H5 首页，确认页面能访问。
2. 打开后台，确认能登录。
3. 检查 API 健康状态。
4. 检查 MySQL 是否正常。
5. 检查服务器磁盘空间。
6. 检查最近是否有大量错误日志。

常用命令：

```bash
curl -i https://rd.chaimen666.com/api/health/ready
df -h
pm2 list
```

如果用 nohup 启动 API，查看日志：

```bash
tail -n 100 /www/wwwroot/rd.chaimen666.com/runtime/api.log
```

如果用 PM2 启动 API，查看日志：

```bash
pm2 logs activity-api --lines 100
```

### 每周检查

1. 检查数据库备份是否正常生成。
2. 检查 Nginx 错误日志。
3. 检查后台登录日志和操作日志。
4. 检查上传目录是否过大。
5. 检查服务器 CPU、内存、磁盘占用。

Nginx 日志位置：

```text
/www/wwwlogs/rd.chaimen666.com.log
/www/wwwlogs/rd.chaimen666.com.error.log
```

查看错误日志：

```bash
tail -n 100 /www/wwwlogs/rd.chaimen666.com.error.log
```

检查上传目录大小：

```bash
du -sh /www/wwwroot/rd.chaimen666.com/uploads
```

如果上传目录实际在 API 或其他目录，以服务器实际目录为准。

### 每月检查

1. 检查 SSL 证书有效期。
2. 做一次数据库备份恢复演练。
3. 检查管理员账号，禁用不用的账号。
4. 检查微信公众平台服务器域名是否仍正确。
5. 检查短信、支付、存储等第三方服务余额和状态。

查看证书有效期：

```bash
echo | openssl s_client -servername rd.chaimen666.com -connect rd.chaimen666.com:443 2>/dev/null | openssl x509 -noout -dates
```

## 九、数据库备份

项目里已有备份脚本：

```bash
cd /www/wwwroot/rd.chaimen666.com
npm run db:backup
```

清理过期备份：

```bash
npm run db:prune-backups
```

恢复备份前一定要谨慎，先确认备份文件和目标数据库。

恢复命令格式：

```bash
RESTORE_CONFIRM=activity_registration BACKUP_FILE=backups/mysql/activity_registration-YYYYMMDD-HHMMSS.sql.gz npm run db:restore
```

恢复生产库前，建议先把当前数据库再备份一份。

## 十、更新代码或重新发布后要做什么

如果只是改后台业务数据，不需要重新构建。

如果修改了代码、环境变量、域名、前端配置，需要按情况重新构建。

### 1. 构建 API

```bash
cd /www/wwwroot/rd.chaimen666.com
npm --prefix apps/api run build
```

然后重启 API：

```bash
pm2 restart activity-api
```

如果不是 PM2：

```bash
pkill -f 'node apps/api/dist/main.js'
nohup node apps/api/dist/main.js > runtime/api.log 2>&1 &
```

### 2. 构建后台

```bash
cd /www/wwwroot/rd.chaimen666.com
npm --prefix apps/admin run build
```

### 3. 构建 H5

```bash
cd /www/wwwroot/rd.chaimen666.com
npm --prefix apps/mobile run build:h5
```

### 4. 构建微信小程序

```bash
cd /www/wwwroot/rd.chaimen666.com
npm --prefix apps/mobile run build:mp-weixin
```

然后用微信开发者工具打开：

```text
apps/mobile/dist/build/mp-weixin
```

上传并提交审核。

## 十一、哪些东西不要随便删

不要随便删除：

```text
/www/wwwroot/rd.chaimen666.com/apps/api/.env
/www/wwwroot/rd.chaimen666.com/.env
/www/wwwroot/rd.chaimen666.com/deploy/.env.production
/www/wwwroot/rd.chaimen666.com/runtime/install.lock
/www/wwwroot/rd.chaimen666.com/uploads
/www/wwwroot/rd.chaimen666.com/backups
```

这些文件和目录可能包含：

1. 数据库连接信息。
2. 登录密钥。
3. 安装锁。
4. 用户上传图片。
5. 数据库备份。

删除前必须先备份。

## 十二、常见故障处理

### 1. H5 能打开，后台能打开，但登录失败

优先检查 API：

```bash
curl -i https://rd.chaimen666.com/api/health/ready
pm2 logs activity-api --lines 100
```

### 2. 访问域名出现宝塔默认页

说明网站目录或 Nginx `root` 不对。

应设置为：

```text
/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5
```

### 3. 后台访问 404

检查 Nginx `/admin/` 的 `alias`：

```nginx
alias /www/wwwroot/rd.chaimen666.com/apps/admin/dist/;
```

不要写成 H5 目录下面的 `apps/admin/dist`。

### 4. 小程序请求失败

检查：

1. 微信公众平台合法域名是否有 `https://rd.chaimen666.com`。
2. API 是否启动。
3. 小程序构建时接口地址是否为线上域名。
4. SSL 证书是否正常。

### 5. 上传图片失败

检查 Nginx 是否保留：

```nginx
location ^~ /uploads/ {
    proxy_pass http://127.0.0.1:3000/uploads/;
}
```

同时检查上传目录权限和磁盘空间：

```bash
df -h
```

## 十三、建议最终使用的稳定运行方式

建议生产服务器使用：

```text
Nginx：宝塔管理，开机自启
MySQL：宝塔管理，开机自启
Node API：PM2 管理，开机自启
```

API 首次用 PM2 启动后执行：

```bash
pm2 startup
pm2 save
```

以后服务器重启后，优先检查：

```bash
pm2 list
curl -i https://rd.chaimen666.com/api/health/ready
```

只要 API、MySQL、Nginx 都正常，H5、后台、小程序基本就能正常运行。
