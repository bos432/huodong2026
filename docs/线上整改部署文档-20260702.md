把本轮整改成果发布到 `https://rd.chaimen666.com`，并完成短信实发、H5 报名闭环、线下收款、签到核销、多角色权限、弹窗、广告、小程序体验版和三端版本一致性验收。

- 生产 `/version.json` 与 `/admin/version.json` 当前返回 HTML fallback，发布时必须补齐真实 JSON。

## 2. 发布前准备

1. 记录发布信息：负责人、发布时间、Git commit、镜像 tag、Admin/H5 buildTime、回滚版本。
2. 备份生产数据库，记录备份文件名和恢复命令。
3. 确认 `git status` 中本轮变更均已评审，未混入无关临时文件。
4. 执行迁移预览：

```powershell
npm --prefix apps/api run migration:show
```

5. 检查生产 `.env`，不得保留 `localhost`、示例域名、默认密钥、占位短信参数。
6. 确认正式运营支付仍使用线下收款，真实微信/支付宝支付和商城真实支付保持关闭，除非单独预发验收通过。

```env

PUBLIC_API_ORIGIN=https://rd.chaimen666.com/api
```

## 4. 构建命令

```powershell
$env:BUILD_COMMIT=(git rev-parse --short HEAD)
$env:BUILD_TIME=(Get-Date).ToUniversalTime().ToString("o")
$env:VITE_BUILD_COMMIT=$env:BUILD_COMMIT
$env:VITE_BUILD_TIME=$env:BUILD_TIME

npm --prefix apps/api run build
npm --prefix apps/admin run build
npm --prefix apps/mobile run build:h5

$env:VITE_API_BASE="https://rd.chaimen666.com/api"
$env:VITE_DEFAULT_TENANT_CODE="qiwai-showcase"
npm --prefix apps/mobile run build:mp-weixin
```

## 7. 回滚方案

触发条件：

- `/api/health/ready` 非 ready。
- 真实短信不可用导致新手机号无法登录。
- 报名、收款、核销任一 P0/P1 阻断。
- migration 导致关键数据异常。
- Admin/H5 静态包不可访问或版本 JSON 缺失。

步骤：

1. 停止新流量或切维护页。
2. 恢复上一版 API 镜像或构建产物。
3. 恢复上一版 Admin/H5 静态包和 `version.json`。
4. 如 migration 不兼容，按发布前备份恢复数据库。
5. reload Nginx，等待 `/api/health/ready`。
6. 复跑登录、报名详情、后台登录最小冒烟。
