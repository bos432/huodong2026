# 线上部署结构与发布说明

本文用于说明当前 `rd.chaimen666.com` 线上环境的真实代码结构、Nginx 映射、构建产物目录和发布检查方法。遇到“代码已拉取但线上页面没变”“后台配置正确但 H5 仍显示旧文案”时，先按本文排查。

## 1. 项目结构

当前仓库是一个 monorepo：

```text
apps/api/       NestJS 后端 API，PM2 运行 apps/api/dist/main.js
apps/admin/     Vue 后台，构建产物在 apps/admin/dist
apps/mobile/    uni-app H5/小程序，H5 构建产物在 apps/mobile/dist/build/h5
packages/shared 公共类型和工具
scripts/        构建、发布、seed、smoke、preflight、备份脚本
docs/           开发、部署、验收和运维文档
deploy/         Docker/Nginx/env 模板
```

## 2. 当前线上 Nginx 映射

宝塔/Nginx 当前配置是“直接服务构建产物目录”，不是把构建产物再复制到项目根目录：

```nginx
server_name rd.chaimen666.com;

root /www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5;

location ^~ /admin/ {
    alias /www/wwwroot/rd.chaimen666.com/apps/admin/dist/;
    try_files $uri $uri/ /admin/index.html;
}

location ^~ /api/ {
    proxy_pass http://127.0.0.1:3000/api/;
}

location ^~ /uploads/ {
    proxy_pass http://127.0.0.1:3000/uploads/;
}

location / {
    try_files $uri $uri/ /index.html;
}
```

因此线上路径对应关系是：

| 访问地址 | 服务来源 | 更新方式 |
| --- | --- | --- |
| `https://rd.chaimen666.com/` | `apps/mobile/dist/build/h5/index.html` | `npm --prefix apps/mobile run build:h5` |
| `https://rd.chaimen666.com/assets/*` | `apps/mobile/dist/build/h5/assets/*` | H5 构建生成 |
| `https://rd.chaimen666.com/admin/` | `apps/admin/dist/index.html` | `npm --prefix apps/admin run build` |
| `https://rd.chaimen666.com/admin/assets/*` | `apps/admin/dist/assets/*` | Admin 构建生成 |
| `https://rd.chaimen666.com/api/*` | PM2 `activity-api` -> `127.0.0.1:3000/api/*` | API build/restart |
| `https://rd.chaimen666.com/uploads/*` | API 静态上传代理 | API/Nginx 配置 |

## 3. 发布方式

当前线上是宝塔 + Nginx + PM2 直出部署。推荐发布命令：

```bash
cd /www/wwwroot/rd.chaimen666.com
set -e

git fetch origin
git pull --ff-only origin feature/qiwai-ui-experiment
git rev-parse --short HEAD

npm --prefix apps/admin run build
npm --prefix apps/mobile run build:h5

WEBROOT=/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5 \
ADMIN_WEBROOT=/www/wwwroot/rd.chaimen666.com/apps/admin/dist \
npm run publish:webroot

grep -o 'assets/index-[^"]*\.js' apps/mobile/dist/build/h5/index.html
grep -R "七维书院\|七维文化\|七维\|奇外\|电召" -n apps/mobile/dist/build/h5 apps/admin/dist 2>/dev/null || echo "OK 没有旧品牌残留"

/www/server/nginx/sbin/nginx -t && /www/server/nginx/sbin/nginx -s reload
```

说明：

- H5 构建会直接写入 Nginx 当前 `root`。
- Admin 构建会直接写入 Nginx 当前 `/admin/` alias。
- `npm run publish:webroot` 在当前线上结构中主要用于识别和提示“构建产物已经是 Nginx 服务目录”；它也保留了另一种部署模式：把 H5/Admin 构建产物复制到独立 webroot。
- 如果只改 H5/Admin，不一定要重启 PM2；如果要更新 `/api/health` release commit 或后端代码，需要重启 PM2。

API release 信息更新：

```bash
PM2=/www/server/nodejs/v22.22.3/lib/node_modules/pm2/bin/pm2

export NODE_ENV=production
export DB_HOST=127.0.0.1
export DB_PORT=3306
export DB_DATABASE=$(awk -F= '/^DB_DATABASE=/{print $2}' apps/api/.env | tail -n1)
export DB_USERNAME=$(awk -F= '/^DB_USERNAME=/{print $2}' apps/api/.env | tail -n1)
export DB_PASSWORD=$(awk -F= '/^DB_PASSWORD=/{print $2}' apps/api/.env | tail -n1)
export BUILD_COMMIT=$(git rev-parse --short HEAD)
export BUILD_TIME=$(date -Iseconds)
export PUBLIC_H5_ORIGIN=https://rd.chaimen666.com
export PUBLIC_ADMIN_ORIGIN=https://rd.chaimen666.com/admin
export PUBLIC_API_ORIGIN=https://rd.chaimen666.com
export CORS_ORIGIN=https://rd.chaimen666.com
export REAL_PAYMENT_ENABLED=false
export PAYMENT_SANDBOX_ENABLED=false

$PM2 restart activity-api --update-env
$PM2 save

API_READY_URL=https://rd.chaimen666.com/api/health/ready npm run wait:api-ready
```

`wait:api-ready` 会轮询 `/api/health/ready`，直到返回 `ready=true` 或超时。PM2 刚重启后的 1-5 秒内，Nginx 可能短暂返回 `502 Bad Gateway`；发布时应等待该脚本通过后再继续执行商城、首页和 smoke 验证。

## 4. 旧标题和旧品牌残留排查

如果后台品牌已经改成 `慢π`，但 H5 顶部或浏览器标题仍显示旧文案，先不要只查后台配置。按顺序判断：

### 4.1 检查外网 HTML 引用的 H5 主包

```bash
curl -s "https://rd.chaimen666.com/?t=$(date +%s)" | grep -E 'assets/index-|七维|慢π' || true
```

如果仍看到：

```text
assets/index-D6hAU5Ez.js
```

说明外网还在加载旧 H5 构建产物，问题在静态构建/发布层，不在数据库品牌配置。

### 4.2 检查 Nginx root 里的 index

```bash
grep -o 'assets/index-[^"]*\.js' apps/mobile/dist/build/h5/index.html
```

这个 hash 应该和外网 `curl` 看到的一致。如果本地构建目录是新 hash，但外网还是旧 hash，说明 Nginx root、缓存或配置没有指向该目录。

### 4.3 检查旧品牌词

```bash
grep -R "七维书院\|七维文化\|七维\|奇外\|电召" -n apps/mobile/dist/build/h5 apps/admin/dist 2>/dev/null || echo "OK 没有旧品牌残留"
```

如果这里还有命中，说明构建产物里仍有旧默认文案，需要回到源码或 seed 数据修正。

如果这里没有命中，但浏览器仍显示旧文案，通常是：

- 外网 `index.html` 仍引用旧 hash。
- 浏览器缓存旧资源。
- Nginx root/alias 指向了另一个目录。
- CDN 或上层缓存未刷新。

### 4.4 检查 API release 和配置

```bash
curl -i https://rd.chaimen666.com/api/health/ready
curl -s "https://rd.chaimen666.com/api/public/homepage?tenantCode=qiwai-showcase" | grep -E "慢π|七维" || true
```

如果 API 已经返回 `慢π`，但 H5 静态包仍是旧 hash，结论仍然是静态文件未更新。

## 5. 构建清理脚本注意事项

H5 构建前会执行：

```bash
node ../../scripts/clean-mobile-h5-dist.mjs
```

由于线上 Nginx 直接服务 `apps/mobile/dist/build/h5`，该脚本不能删除 `h5` 根目录本身，只能清空其内部文件。否则 Nginx 短时间内可能找不到 root 目录。

脚本也不能依赖 Node `fs.rm(..., { recursive: true })` 的内部递归判断。服务器 Node v20 曾出现把 `.css` 文件当目录 `scandir`、或对 `assets` 目录执行 `unlink` 的差异行为。因此脚本应使用明确的 `readdir -> unlink/rmdir` 逐层删除逻辑。

如果服务器构建时报：

```text
EPERM: operation not permitted, unlink '.../apps/mobile/dist/build/h5/assets/xxx.css'
```

优先判断为旧静态文件或父目录存在 Linux 不可变/追加属性、权限异常，导致 root 也不能删除。当前清理脚本会在 `EPERM/EACCES` 时自动尝试：

- `chattr -i -a 当前文件`
- `chattr -i -a 父目录`
- `chmod` 放宽当前文件或目录权限
- 重试删除

如果自动解锁后仍失败，在服务器手动执行：

```bash
cd /www/wwwroot/rd.chaimen666.com

lsattr apps/mobile/dist/build/h5/assets/AdminBottomNav-kku8XPez.css 2>/dev/null || true
chattr -i -a apps/mobile/dist/build/h5/assets/AdminBottomNav-kku8XPez.css 2>/dev/null || true
chattr -R -i -a apps/mobile/dist/build/h5 2>/dev/null || true
chmod -R u+rwX apps/mobile/dist/build/h5 apps/admin/dist 2>/dev/null || true

npm --prefix apps/mobile run build:h5
```

不要用 `rm -rf /www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5` 删除 H5 根目录；线上 Nginx 正在把它作为站点根目录服务。需要清理时，只清理目录内部文件，或者使用项目自带的 `clean-mobile-h5-dist.mjs`。

## 6. 判断问题在哪一层

| 现象 | 优先判断 |
| --- | --- |
| API health commit 已更新，但页面仍旧 | H5/Admin 静态构建未成功或 Nginx 仍指旧目录 |
| 外网 HTML 仍引用旧 hash | H5 构建未成功、Nginx root 指向旧目录或缓存未刷新 |
| 构建目录 index 是新 hash，外网是旧 hash | Nginx root/缓存问题 |
| 构建目录仍含旧品牌词 | 源码默认文案或 seed 数据仍有旧词 |
| H5 页面能打开但列表/登录失败 | API/PM2/数据库/Nginx `/api/` 代理问题 |
| 后台 `/admin/` 404 或空白 | Admin 构建产物、Nginx alias 或 SPA fallback 问题 |

## 7. 本次旧标题问题结论

本次 `七维书院` 标题残留的实际路径是：

1. 后台和公开接口已经返回 `慢π`，数据层基本正确。
2. 外网 `index.html` 仍引用旧主包 `assets/index-D6hAU5Ez.js`。
3. Nginx 实际 root 是 `apps/mobile/dist/build/h5`。
4. H5 构建在 `prebuild:h5` 清理阶段中断，导致新静态包没有生成。
5. 修复重点应放在 `scripts/clean-mobile-h5-dist.mjs` 和构建发布流程，而不是继续改后台品牌设置。
