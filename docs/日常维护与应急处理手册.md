# 日常维护与应急处理手册

本文档用于日常维护 `rd.chaimen666.com` 活动报名系统，以及出现 502、接口失败、后台打不开、服务器重启后服务未恢复等问题时的应急处理。

当前线上地址：

```text
H5：首页/用户端
https://rd.chaimen666.com/#/pages/index/index

后台管理端：
https://rd.chaimen666.com/admin/

API 健康检查：
https://rd.chaimen666.com/api/health/ready
```

服务器项目目录：

```bash
/www/wwwroot/rd.chaimen666.com
```

当前 Node / PM2 路径：

```bash
Node 目录：/www/server/nodejs/v22.22.3
PM2 命令：/www/server/nodejs/v22.22.3/bin/pm2
```

## 一、系统由哪些服务组成

线上运行主要依赖 4 个部分：

| 服务 | 作用 | 异常表现 |
| --- | --- | --- |
| Nginx | 负责域名访问、HTTPS、H5、后台、API 反向代理 | 网站打不开、404、502 |
| MySQL | 保存活动、用户、订单、报名、后台设置等数据 | 登录失败、接口 500、健康检查 database down |
| Node API | 后端接口服务，监听 `3000` 端口 | 后台登录 502、H5 列表加载失败、报名失败 |
| PM2 | 管理 Node API 进程 | API 停掉后不能自动恢复 |

日常维护时，优先确认这几个服务是否正常。

## 二、常用命令分别是什么作用

### 1. 查看 API 是否被 PM2 托管

```bash
/www/server/nodejs/v22.22.3/bin/pm2 list
```

作用：

查看 `activity-api` 是否在线。

正常结果：

```text
activity-api online
```

什么时候执行：

- 每天巡检时执行一次。
- 后台登录 502 时执行。
- 服务器重启后执行。
- 修改后端代码并重启后执行。

如果看到 `errored`、`stopped`、没有 `activity-api`，说明 API 没有正常运行，需要重启或重新启动。

### 2. 检查公网 API 是否正常

```bash
curl -i https://rd.chaimen666.com/api/health/ready
```

作用：

从公网域名检查 API、数据库、配置是否可用。

正常结果：

```text
HTTP/2 200
```

并且返回内容里应该有：

```json
"ready": true,
"api": "up",
"database": "up"
```

什么时候执行：

- 每天巡检。
- 后台或 H5 报错时。
- Nginx 配置修改后。
- PM2 重启 API 后。
- 宝塔计划任务执行后。

如果不是 200，说明公网访问链路有问题，可能是 Nginx、API 或数据库异常。

### 3. 检查本机 API 是否正常

```bash
curl -i http://127.0.0.1:3000/api/health/ready
```

作用：

绕过公网域名和 Nginx，直接检查服务器本机 `3000` 端口的 API。

什么时候执行：

- 公网 API 502 时。
- 判断问题是 Nginx 还是 Node API 时。

判断方法：

| 本机 API | 公网 API | 说明 |
| --- | --- | --- |
| 正常 | 正常 | 服务正常 |
| 正常 | 异常 | 多半是 Nginx 配置或 SSL/代理问题 |
| 异常 | 异常 | 多半是 API 没启动或 API 启动失败 |

### 4. 重启 API

```bash
/www/server/nodejs/v22.22.3/bin/pm2 restart activity-api
```

作用：

重启后端接口服务。

什么时候执行：

- 修改了 `apps/api/.env` 后。
- 修改了后端代码并重新构建后。
- API 卡住、接口持续报错时。
- 健康检查异常，但 PM2 仍显示 online 时。

重启后必须验证：

```bash
/www/server/nodejs/v22.22.3/bin/pm2 list
curl -i https://rd.chaimen666.com/api/health/ready
```

### 5. 查看 API 日志

```bash
/www/server/nodejs/v22.22.3/bin/pm2 logs activity-api --lines 100
```

作用：

查看最近 100 行 API 日志，用于排查报错原因。

什么时候执行：

- 后台登录失败。
- 接口 500。
- 502 反复出现。
- 微信支付、短信、上传图片异常。
- PM2 显示 API 一直重启。

退出日志实时查看：

```text
Ctrl + C
```

### 6. 查看 Nginx 配置是否正确

```bash
nginx -t
```

作用：

检查 Nginx 配置语法。

什么时候执行：

- 修改宝塔网站配置后。
- 修改 `/api/`、`/admin/`、`/uploads/` 代理后。
- 网站出现 404、502、重定向异常时。

正常结果：

```text
syntax is ok
test is successful
```

### 7. 重载 Nginx

```bash
nginx -s reload
```

作用：

让新的 Nginx 配置生效，不重启服务器。

什么时候执行：

- `nginx -t` 检查成功后。
- 修改网站配置后。

### 8. 查看磁盘空间

```bash
df -h
```

作用：

查看服务器磁盘是否快满。

什么时候执行：

- 每周巡检。
- 上传图片失败。
- 数据库异常。
- 日志写不进去。
- 宝塔提示磁盘空间不足。

重点看 `/`、`/www` 所在分区，如果使用率超过 85%，需要清理日志、备份或无用文件。

### 9. 查看 Nginx 错误日志

```bash
tail -n 100 /www/wwwlogs/rd.chaimen666.com.error.log
```

作用：

查看网站访问层面的错误，例如代理失败、文件不存在、权限问题。

什么时候执行：

- 域名打不开。
- 502。
- 后台或 H5 静态资源 404。
- 上传图片访问失败。

## 六、服务器重启后怎么处理

服务器重启后，按这个顺序检查。

### 1. 检查 API 是否自动恢复

```bash
/www/server/nodejs/v22.22.3/bin/pm2 list
```

如果显示：

```text
activity-api online
```

继续检查健康接口：

```bash
curl -i https://rd.chaimen666.com/api/health/ready
```

### 2. 如果 API 没起来，手动恢复

```bash
cd /www/wwwroot/rd.chaimen666.com
/www/server/nodejs/v22.22.3/bin/pm2 resurrect
/www/server/nodejs/v22.22.3/bin/pm2 list
```

如果 `resurrect` 后还是没有 `activity-api`：

```bash
cd /www/wwwroot/rd.chaimen666.com
/www/server/nodejs/v22.22.3/bin/pm2 start apps/api/dist/main.js --name activity-api --cwd /www/wwwroot/rd.chaimen666.com
/www/server/nodejs/v22.22.3/bin/pm2 save
```

最后验证：

```bash
curl -i https://rd.chaimen666.com/api/health/ready
```

### 3. 宝塔计划任务建议

由于当前服务器 `systemd` 和 `crontab` 写入受限，建议在宝塔面板配置计划任务。

任务类型：

```text
Shell 脚本
```

执行周期：

```text
每 5 分钟
```

脚本内容：

```bash
cd /www/wwwroot/rd.chaimen666.com

/www/server/nodejs/v22.22.3/bin/pm2 resurrect

/www/server/nodejs/v22.22.3/bin/pm2 list | grep activity-api >/dev/null 2>&1

if [ $? -ne 0 ]; then
  /www/server/nodejs/v22.22.3/bin/pm2 start apps/api/dist/main.js --name activity-api --cwd /www/wwwroot/rd.chaimen666.com
  /www/server/nodejs/v22.22.3/bin/pm2 save
fi
```

这个任务的作用：

- 如果 API 正常运行，不重复启动。
- 如果 API 没有运行，自动拉起。
- 如果服务器重启后 PM2 没恢复，最多 5 分钟内自动恢复。

## 七、常见故障应急处理

## 故障 1：后台登录报 502

表现：

```text
https://rd.chaimen666.com/admin/login
登录时报 502
```

最常见原因：

API 没启动，Nginx 代理不到 `127.0.0.1:3000`。

处理步骤：

```bash
/www/server/nodejs/v22.22.3/bin/pm2 list
curl -i http://127.0.0.1:3000/api/health/ready
curl -i https://rd.chaimen666.com/api/health/ready
```

如果本机 API 不通，执行：

```bash
cd /www/wwwroot/rd.chaimen666.com
/www/server/nodejs/v22.22.3/bin/pm2 restart activity-api
```

如果提示找不到 `activity-api`：

```bash
cd /www/wwwroot/rd.chaimen666.com
/www/server/nodejs/v22.22.3/bin/pm2 start apps/api/dist/main.js --name activity-api --cwd /www/wwwroot/rd.chaimen666.com
/www/server/nodejs/v22.22.3/bin/pm2 save
```

然后验证：

```bash
curl -i https://rd.chaimen666.com/api/health/ready
```

## 故障 2：API 一启动就退出

表现：

```bash
pm2 list
```

看到：

```text
errored
```

或 `restart` 次数一直增加。

处理步骤：

```bash
/www/server/nodejs/v22.22.3/bin/pm2 logs activity-api --lines 150
```

常见原因：

| 日志关键词 | 可能原因 | 处理方式 |
| --- | --- | --- |
| `Invalid production configuration` | 生产配置缺失 | 检查 `apps/api/.env` |
| `Access denied for user` | 数据库账号或密码错误 | 检查数据库配置 |
| `ECONNREFUSED 127.0.0.1:3306` | MySQL 未启动 | 启动 MySQL |
| `Cannot find module` | 依赖或构建缺失 | 重新安装依赖并构建 |
| `EADDRINUSE 3000` | 3000 端口被占用 | 查找并处理占用进程 |

检查 3000 端口：

```bash
ss -lntp | grep 3000
```

## 故障 3：H5 首页打开是宝塔默认页

原因：

宝塔网站目录或 Nginx `root` 配错了。

正确网站目录：

```text
/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5
```

正确 Nginx 根目录：

```nginx
root /www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5;
```

并且需要：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

修改后执行：

```bash
nginx -t
nginx -s reload
```

## 故障 4：后台页面 404 或打不开

原因：

`/admin/` 的 Nginx alias 配错。

正确配置：

```nginx
location = /admin {
    return 301 /admin/;
}

location ^~ /admin/ {
    alias /www/wwwroot/rd.chaimen666.com/apps/admin/dist/;
    index index.html;
    try_files $uri $uri/ /admin/index.html;
}
```

检查后台构建文件是否存在：

```bash
ls -l /www/wwwroot/rd.chaimen666.com/apps/admin/dist/index.html
```

如果不存在，重新构建：

```bash
cd /www/wwwroot/rd.chaimen666.com
npm --prefix apps/admin run build
```

## 故障 5：H5 或后台页面能打开，但数据加载失败

可能原因：

- API 没启动。
- `/api/` Nginx 代理错。
- 数据库异常。
- 前端构建时 API 地址错误。

处理步骤：

```bash
curl -i http://127.0.0.1:3000/api/health/ready
curl -i https://rd.chaimen666.com/api/health/ready
/www/server/nodejs/v22.22.3/bin/pm2 logs activity-api --lines 100
```

如果本机正常但公网异常，检查 Nginx：

```nginx
location ^~ /api/ {
    proxy_pass http://127.0.0.1:3000/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## 故障 6：数据库异常

健康检查可能显示：

```json
"database": "down"
```

处理步骤：

先在宝塔面板检查 MySQL 是否运行：

```text
宝塔 -> 软件商店 -> MySQL -> 状态
```

命令行可尝试：

```bash
systemctl status mysqld
systemctl status mysql
```

如果 MySQL 没启动，优先在宝塔面板启动。

如果日志显示数据库账号错误，检查：

```bash
cd /www/wwwroot/rd.chaimen666.com
grep -n "DB_HOST\|DB_PORT\|DB_USERNAME\|DB_DATABASE" .env apps/api/.env
```

不要在公共聊天或截图里泄露 `DB_PASSWORD`。

修改数据库配置后，需要重启 API：

```bash
/www/server/nodejs/v22.22.3/bin/pm2 restart activity-api
```

## 故障 7：上传图片失败

可能原因：

- API 异常。
- 上传目录权限异常。
- Nginx `/uploads/` 配置缺失。
- 磁盘满了。

处理步骤：

```bash
df -h
curl -i https://rd.chaimen666.com/api/health/ready
tail -n 100 /www/wwwlogs/rd.chaimen666.com.error.log
```

Nginx 应有：

```nginx
location ^~ /uploads/ {
    proxy_pass http://127.0.0.1:3000/uploads/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## 故障 8：小程序请求失败

检查顺序：

1. API 健康检查是否正常。
2. 微信公众平台合法域名是否配置 `https://rd.chaimen666.com`。
3. 小程序是否重新构建并上传。
4. 小程序版本是否已经提交审核并发布。
5. SSL 证书是否有效。

命令检查：

```bash
curl -i https://rd.chaimen666.com/api/health/ready
```

微信公众平台需要配置：

```text
request 合法域名：https://rd.chaimen666.com
uploadFile 合法域名：https://rd.chaimen666.com
downloadFile 合法域名：https://rd.chaimen666.com
```

## 十一、维护判断标准

系统正常的最低标准：

```text
PM2 中 activity-api 为 online
https://rd.chaimen666.com/api/health/ready 返回 200
H5 首页能打开
后台登录页能打开
后台能正常登录
活动列表能加载
```

上线稳定运行建议：

- 宝塔计划任务每 5 分钟检查并恢复 API。
- 每天检查一次健康接口。
- 每周检查一次日志和数据库备份。
- 每月检查一次 SSL、管理员账号和第三方服务配置。
