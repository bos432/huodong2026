# 后台系统设置说明

本文档用于说明后台「系统设置」里的域名配置、部署配置、保存设置和生效方式。

后台地址：

```text
https://rd.chaimen666.com/admin/
```

## 一、后台里的域名能不能设置

可以设置，但要分清两类配置：

1. 后台页面里的「运营设置」通常会保存到数据库，适合配置业务信息。
2. 后台页面里的「部署配置」一般用于生成 `deploy/.env.production` 内容，不一定会自动写入服务器环境文件。

从你截图里的提示看：

```text
部署级配置不会保存到数据库。这里生成 deploy/.env.production 内容，部署到服务器后重启 API 生效。
```

意思是：这里的域名配置主要是给部署环境变量用的。保存或生成后，还需要把生成的 `.env.production` 内容放到服务器对应文件，并重启 API 服务，才能真正影响后端运行。

## 二、域名应该怎么填

你现在是 H5、后台、API 都走同一个域名 `rd.chaimen666.com`，只是路径不同。

推荐填写：

| 配置项 | 应填写 |
| --- | --- |
| H5 域名 `PUBLIC_H5_ORIGIN` | `https://rd.chaimen666.com` |
| 后台域名 `PUBLIC_ADMIN_ORIGIN` | `https://rd.chaimen666.com` |
| API 域名 `PUBLIC_API_ORIGIN` | `https://rd.chaimen666.com` |
| 上传目录 `UPLOAD_DIR` | `uploads` |

不要填写：

```text
https://rd.chaimen666.com/admin/
https://rd.chaimen666.com/api/
```

这里一般填「源站域名」，也就是协议加域名，不带后面的路径。

## 三、为什么后台域名也填主域名

因为你的 Nginx 配置是：

```text
H5：首页访问 https://rd.chaimen666.com/
后台：访问 https://rd.chaimen666.com/admin/
API：访问 https://rd.chaimen666.com/api/
```

这三个入口共用同一个域名，所以后台里的三个域名都可以填：

```text
https://rd.chaimen666.com
```

后台真正访问地址仍然是：

```text
https://rd.chaimen666.com/admin/
```

API 真正请求地址仍然是：

```text
https://rd.chaimen666.com/api
```

## 四、后台建议填写示例

在「系统设置 -> 部署配置」里建议填写：

```text
版本号：0.1.0
发布提交：可以先留空
H5 域名：https://rd.chaimen666.com
后台域名：https://rd.chaimen666.com
API 域名：https://rd.chaimen666.com
上传目录：uploads
```

数据库密码、JWT 密钥、H5 登录密钥等安全项不要随便改。

如果当前项目已经能正常登录后台和访问小程序，安全密钥类配置尽量保持原样。

## 五、保存设置后会不会马上生效

要看配置类型。

| 配置类型 | 是否马上生效 |
| --- | --- |
| 运营设置、业务开关、页面内容 | 通常保存后马上生效 |
| H5 域名、后台域名、API 域名 | 通常需要更新 `.env.production` 并重启 API |
| 数据库密码、JWT 密钥、安全密钥 | 需要谨慎修改，改错会导致服务异常 |
| 小程序 AppID、接口地址 | 通常需要重新打包小程序 |

## 六、部署配置正确生效流程

如果你在后台修改了「部署配置」里的域名，建议按下面流程处理。

### 1. 在后台填写域名

填写：

```text
H5 域名：https://rd.chaimen666.com
后台域名：https://rd.chaimen666.com
API 域名：https://rd.chaimen666.com
```

点击：

```text
保存设置
```

### 2. 复制或下载生成的 `.env.production`

右侧有：

```text
生成的 .env.production
```

可以点击：

```text
复制
```

或：

```text
下载
```

### 3. 放到服务器部署文件

服务器目标文件通常是：

```text
/www/wwwroot/rd.chaimen666.com/deploy/.env.production
```

如果项目实际路径不同，以服务器项目目录为准。

### 4. 重启 API 服务

更新环境变量后，需要重启后端 API。

如果是 PM2 管理，常见命令：

```bash
pm2 list
pm2 restart all
```

如果知道具体服务名，可以只重启对应服务：

```bash
pm2 restart 服务名
```

### 5. 重新打包前端

如果前端、小程序构建时读取了这些环境变量，修改域名后还需要重新打包。

H5 打包：

```bash
cd /www/wwwroot/rd.chaimen666.com
npm --prefix apps/mobile run build:h5
```

后台打包：

```bash
cd /www/wwwroot/rd.chaimen666.com
npm --prefix apps/admin run build
```

小程序打包：

```bash
cd /www/wwwroot/rd.chaimen666.com
npm --prefix apps/mobile run build:mp-weixin
```

## 七、当前 Nginx 配置对应关系

当前建议的 Nginx 访问关系是：

```text
https://rd.chaimen666.com/        -> H5
https://rd.chaimen666.com/admin/  -> 后台
https://rd.chaimen666.com/api/    -> 后端 API
https://rd.chaimen666.com/uploads/ -> 上传文件
```

因此后台里不要把 H5 域名写成 H5 文件目录，也不要写服务器路径。

错误示例：

```text
/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5
```

正确示例：

```text
https://rd.chaimen666.com
```

后台配置里填的是浏览器访问域名，不是服务器文件目录。

## 八、上线前检查清单

上线前建议检查：

1. H5 能打开：`https://rd.chaimen666.com/#/pages/index/index`
2. 后台能打开：`https://rd.chaimen666.com/admin/`
3. API 能访问：`https://rd.chaimen666.com/api`
4. 上传目录能正常读写。
5. 微信公众平台已配置合法域名：`https://rd.chaimen666.com`
6. 小程序中没有 `localhost`、`127.0.0.1`、内网 IP。
7. 后台部署配置里的三个域名都使用 HTTPS。

## 九、不建议随便修改的项目

以下配置不建议随便改：

```text
DB_PASSWORD
MYSQL_ROOT_PASSWORD
JWT_SECRET
H5_AUTH_SECRET
TRUST_PROXY
SECURITY_HEADERS_ENABLED
SECURITY_HSTS_ENABLED
```

这些属于数据库连接、安全登录、代理和安全头配置。改错可能导致：

1. 后台无法登录。
2. API 启动失败。
3. 小程序请求失败。
4. 登录状态全部失效。

如果只是为了发布小程序，通常只需要确认域名、AppID、接口地址，不需要改这些安全配置。
