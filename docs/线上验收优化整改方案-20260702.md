## 4. 建议纳入长期流程

1. 发布流水线固定顺序：`git pull`、依赖安装、构建、migration、`publish:webroot`、PM2 重启、Nginx reload、本机 ready、公网 ready。
2. 每次生产 seed 后立即执行 `smoke:online-showcase` 和 `verify-online-showcase-roles.mjs`。
3. 浏览器验收固定保存 `.local-logs/browser-acceptance-*`，测试数据不清理，作为订单、报名、签到追溯证据。
4. 将短信服务商余额、签名模板审核状态、验证码日志纳入日常巡检。
5. 小程序体验版每次发布至少保留二维码、机型、微信版本、登录截图、首页截图、活动详情截图。
