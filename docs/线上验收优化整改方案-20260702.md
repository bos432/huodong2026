## 5. 后续优化建议

1. 将 `npm run browser:online-showcase` 加入发布流水线的人工确认步骤，生产发布后立即复跑。
2. 生产部署统一由同一流水线写入 `BUILD_COMMIT`、`BUILD_TIME`、`VITE_BUILD_COMMIT`、`VITE_BUILD_TIME`。
3. Nginx 明确放行 `/version.json` 与 `/admin/version.json` 静态文件，避免 SPA fallback。
4. 小程序体验版验收记录固定上传到发布附件，至少包含二维码、iOS/Android 机型、微信版本和截图路径。
5. 真实短信开通后补一条“短信服务商日常巡检”到运维手册，覆盖余额、签名模板审核、失败告警和验证码限流日志。
