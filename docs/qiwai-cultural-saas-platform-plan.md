# 七维文化活动与课程 SaaS 平台落地方案

更新时间：2026-06-11

## 1. 项目定位

七维文化第一年不做大而全知识平台，先聚焦：

> 线下文化活动 + 课程报名 + 城市合伙人运营 SaaS。

平台面向传统文化、书法、美育、家庭教育、健康养生、创业技能等领域，支持平台方统一监管，城市合伙人/商家独立运营、独立收款、独立管理活动与会员。

现有活动报名系统继续作为主平台，不建议首年再购买独立知识付费 SaaS 模板。录播课、APP 和复杂知识付费专栏放到后续阶段，根据真实报名、复购和城市复制数据决定是否投入。

## 2. 第一阶段产品范围

### 保留并重点使用

| 能力 | 落地方式 |
| --- | --- |
| H5 活动报名 | 用于公开活动展示、报名、支付、会员转化 |
| 管理后台 | 平台、商家和员工统一从后台运营 |
| 多商家/代理 SaaS | 城市合伙人/机构作为 `Tenant` 租户 |
| 活动发布审核 | 商家提交活动，平台审核后公开 |
| 报名审核权限 | 平台决定商家是否可开启报名审核 |
| 商家独立收款 | 不同商家绑定自己的支付/收款配置 |
| 订单与财务 | 支持订单、退款、对账、导出和财务监管 |
| 会员与积分 | 支持会员分层、积分、复购和活动门槛 |
| 代理结算 | 保留为商家内部渠道或地区代理结算能力 |

### 暂缓投入

| 暂缓项 | 原因 | 建议启动条件 |
| --- | --- | --- |
| 原生 APP | 初期开发和维护成本高 | 月活、复购和课程留存稳定后再做 |
| 大规模录播课系统 | 内容制作压力大，现金流回收慢 | 找到 2-3 个高复购课程方向后再做 |
| 复杂知识付费专栏 | 与当前活动报名主链路不一致 | 活动用户沉淀为稳定会员后再扩展 |
| 七大板块同时铺开 | 师资、内容、运营都会被摊薄 | 先跑通 3 个主方向，再逐步扩展 |
| 多级代理体系 | 权限、分账、结算复杂度高 | 城市合伙人模型稳定后进入二期 |

### 第一阶段内容重点

第一阶段只重点做 3 个内容方向：

1. 国学 / 东方哲学与传统文化
2. 书法 / 美育
3. 家庭教育、健康养生、创业技能中选择一个最容易变现的方向

## 3. 角色与系统菜单映射

### 平台超级管理员

平台超级管理员负责总控和监管：

| 菜单方向 | 主要能力 |
| --- | --- |
| 平台概览 | 查看商家数、活动数、订单金额、待审核活动、退款和异常回调 |
| 商家管理 | 创建城市合伙人/商家，停用商家，维护商家资料和备注 |
| 商家账号 | 给商家创建管理员、运营、财务、签到账号 |
| 权限配置 | 控制活动免审、报名审核权限、收款配置权限 |
| 活动审核 | 审核商家活动，通过、驳回、下架违规活动 |
| 财务监管 | 查看全局订单、流水、退款、对账、商家收款账户 |
| 运营监管 | 查看全局报名、公告、首页装修和运营数据 |
| H5 全局装修 | 配置平台默认首页、活动分页、底部菜单、我的页和内页布局 |
| 系统与安全 | 系统设置、上线体检、操作日志、登录日志、管理员管理 |

### 商家 / 城市合伙人管理员

城市合伙人本质上是商家租户，拥有自己的后台：

| 菜单方向 | 主要能力 |
| --- | --- |
| 商家概览 | 查看本商家活动、报名、订单、收入、待办和平台授权状态 |
| 活动运营 | 新建活动、编辑活动、提交审核、下架活动、管理分类 |
| 首页与公告 | 配置本商家 H5 首页、公告、通知 |
| 报名与现场 | 管理报名、候补、签到核销、报名导出 |
| 票务与财务 | 管理票种、优惠码、订单、退款、财务对账、收款方式 |
| 用户与会员 | 管理会员、积分、等级、用户标签 |
| 商家设置 | 修改商家资料、运营设置、员工账号、操作日志 |

### 商家员工

| 角色 | 可用范围 | 限制 |
| --- | --- | --- |
| 商家运营 | 活动、分类、首页、公告、通知、报名、会员标签 | 不能配置收款方式，不能看完整财务，不能建员工 |
| 商家财务 | 财务看板、订单、退款、对账、收款账户只读或受限编辑 | 不能创建活动，不能改商家权限 |
| 商家签到 | 签到核销、活动列表、报名查询 | 不能看财务，不能审核活动，不能配置收款 |

## 4. 商业模式调整

首年重点收入不以录播课销售为核心，而是围绕线下活动和城市复制：

| 收入来源 | 说明 |
| --- | --- |
| 线下活动报名费 | 沙龙、体验课、读书会、书法课、养生课 |
| 训练营 / 课程报名费 | 7 天、14 天、30 天训练营或专题班 |
| 城市合伙人 / 商家入驻费 | 城市合伙人使用系统、品牌、活动模型和运营 SOP |
| 会员年卡 | 面向稳定用户提供活动优惠、优先报名、会员专属活动 |
| 讲师 / 机构合作分成 | 平台提供系统和流量，讲师提供内容 |
| 企业内训 / 私董会 | 高客单服务，用于提升现金流和品牌势能 |

核心指标：

| 指标 | 用途 |
| --- | --- |
| 单场报名转化率 | 判断活动主题和落地页是否有效 |
| 单场活动毛利 | 判断城市活动是否能复制 |
| 用户复购率 | 判断是否具备会员化基础 |
| 城市合伙人获客成本 | 判断招商模型是否健康 |
| 商家月均 GMV | 判断商家是否值得继续扶持 |
| 退款率 / 投诉率 | 判断课程交付和宣传是否稳健 |
| 讲师分成后平台净收入 | 判断平台真实利润 |

## 5. 合规要求

“玄学”统一调整为：

> 东方哲学与传统文化。

禁止出现：

- 算命
- 改运
- 破灾
- 保证结果
- 预测财富、婚姻、疾病
- 封建迷信式营销

允许方向：

- 易经文化
- 民俗文化
- 节气文化
- 传统哲学
- 人文地理
- 国学经典解读

后台活动审核、公告审核和商家运营培训中都应纳入这些合规口径。

## 6. 18 个月路线图

| 阶段 | 时间 | 目标 | 关键交付 |
| --- | --- | --- | --- |
| 第 1 阶段 | 0-2 个月 | 本地系统稳定可演示，完成首批活动上线 | 完成 SaaS 多商家后台验收；开通 2-3 个测试商家；每商家发布 2-3 个活动；跑通报名、支付、签到、财务、收款账户；输出招商页面和合伙人说明书 |
| 第 2 阶段 | 3-6 个月 | 跑通第一个城市样板 | 招募 3-5 个城市合伙人；每城每月至少 2 场活动；建立会员体系；建立讲师合作分成规则；建立活动复盘、财务对账、退款规则 |
| 第 3 阶段 | 7-12 个月 | 复制城市模型 | 城市合伙人达到 10 个左右；平台活动月 GMV 稳定增长；增加课程包、训练营、企业内训；录播课轻量补充；优化商家经营数据看板 |
| 第 4 阶段 | 13-18 个月 | 从活动平台升级为课程生态 | 根据真实数据决定是否开发 APP；根据高复购方向建设录播课；建立讲师认证和内容审核体系；扩展城市合伙人结算与分润模型 |

## 7. 验收清单

### 业务验收

- 平台超级管理员可创建商家、配置权限、审核活动。
- 商家只能看到自己的活动、报名、订单、会员和财务。
- 商家可配置自己的收款方式，订单支付到对应商家。
- 商家无报名审核权限时，不能开启报名审核。
- 待审核活动不在 H5 公开展示。
- 审核通过后活动可报名和支付。

### 商业验收

- 至少 2-3 个测试商家可独立完成活动发布和报名管理。
- 至少 1 个城市样板能持续每月举办活动。
- 活动复盘可看到报名、付款、签到、退款和收入数据。
- 财务对账能区分平台、商家和代理归属。
- 招商材料能清楚说明合伙人收益、责任、权限和结算规则。

## 8. 执行默认假设

- 第一版继续采用“商家/代理 = Tenant 租户”。
- 城市合伙人本质上就是商家租户。
- 原 `Agent` 保留为商家内部渠道/地区代理能力，放到二期。
- 首年以 H5 + 后台为主，不优先开发 APP。
- 录播课先轻量化，用活动回放或课程包验证需求。
- 先跑通一个城市样板，再复制全国。
