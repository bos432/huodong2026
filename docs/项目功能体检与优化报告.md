# 活动报名系统功能体检与优化报告

检查日期：2026-06-16  
检查视角：平台超级管理员、商家管理员、运营人员、财务人员、签到人员、H5 用户端、手机管理端

## 一、先回答：活动分类在哪里添加？

当前系统已经有“分类管理”功能，页面文件是：

- 后台页面：`apps/admin/src/views/Categories.vue`
- 后台路由：`/admin/categories`
- 后端接口：`GET/POST/PATCH /admin/categories`

但是现在有一个明显的产品入口问题：

- 商家端菜单里有：`商家端 · 活动 -> 分类管理`
- 平台超级管理员菜单里没有“分类管理”
- 后端其实支持平台超级管理员创建“全局分类”

所以你现在在平台超管视角看到活动列表里的“分类”列，却找不到添加分类入口，这是因为“功能存在，但菜单入口缺失”。

### 现在临时怎么添加？

如果你当前登录的是平台超级管理员，可以直接在浏览器打开：

```text
https://rd.chaimen666.com/admin/categories
```

如果页面能打开，就点右上角“新增分类”。  
如果跳回首页，说明前端路由把平台超管挡住了，需要修菜单和路由 scope。

### 建议修复

把“分类管理”同时放到平台端菜单：

```text
平台端 · 活动监管 -> 分类管理
```

同时把路由 `/categories` 的 scope 从 `tenant` 改成 `tenantOrPlatformAdmin`，这样：

- 平台超管维护全局分类
- 商家管理员维护自己商家的分类
- H5 无商家域名时展示全局分类
- H5 有商家域名时展示商家分类

优先级：P0，建议马上修。

## 二、总体结论

系统现在已经不是“简单报名系统”，而是一个正在变成“活动运营 + 商家管理 + 会员资产 + 公益池 + 数据中心 + 文化大使招募”的平台。

当前核心能力基本具备：

- 活动发布、审核、报名、订单、签到
- H5 首页装修、底部菜单、活动详情
- 后台会员管理、余额、积分、公益基金
- 手机管理端、线下收款、人工确认
- 数据中心、渠道码、活动漏斗、复盘
- 文化大使招募落地页和线索管理

但当前最大的问题是：

```text
功能越来越多，但菜单、权限、操作路径、说明文案还没有同步产品化。
```

换句话说，很多能力已经做了，但用户会遇到：

- 不知道入口在哪里
- 不知道这个按钮什么时候用
- 超管和商家角色边界不清
- 后台列表里能看到字段，但找不到维护入口
- 部分功能有配置项，但缺少“下一步怎么做”的引导
- 移动端和后台端能力不完全一致

## 三、P0 必须优先修复的问题

### 1. 平台超管缺少“分类管理”入口

现象：

- 活动列表有“分类”列。
- 筛选器里有“全部分类”。
- 但平台超管菜单没有“分类管理”。

影响：

- 平台无法维护全局分类。
- 新建活动时不知道分类从哪里来。
- 用户会误以为系统功能没做完。

建议：

- 平台端菜单增加：`平台端 · 活动监管 -> 分类管理`
- `/categories` 路由允许平台超管访问
- 分类页面增加说明：`平台分类适用于平台自营活动；商家分类仅适用于对应商家活动`

验收：

- 平台超管能新增“国学、书法、公益、创业、亲子、健康”等全局分类。
- 新建平台活动时能选择这些分类。
- H5 活动列表能看到分类筛选。

### 2. 活动列表操作按钮太多，信息密度过高

现象：

活动列表每行有：

- 预览H5
- 复制链接
- 二维码
- 海报
- 渠道
- 审核记录
- 编辑
- 下架

影响：

- 超管审核活动时，核心动作被淹没。
- 新手不知道哪个按钮是主要操作。
- 表格横向滚动明显，容易误操作。

建议：

- 主按钮只保留 2-3 个：`预览`、`审核/编辑`、`更多`
- 复制链接、二维码、海报、渠道、审核记录放进“更多”下拉菜单
- 不同状态展示不同主按钮：
  - 待审核：审核、预览
  - 报名中：预览、渠道、下架
  - 已结束：复盘、数据

优先级：P0。

### 3. 真实支付能力还不是完整闭环

代码里能看到真实支付相关 adapter 仍有未实现分支：

- 微信/支付宝真实下单 SDK
- 真实支付回调验签
- 真实退款请求和查询
- 服务商账单拉取
- 代理真实打款

现在可稳定使用的更像是：

- 免费报名
- 余额支付
- 线下收款 / 人工确认
- mock / 沙箱能力

影响：

- 如果直接打开微信支付/支付宝，可能会遇到生产不可用或回调不完整。
- 财务对账、退款自动化还不能完全依赖系统。

建议：

- 后台支付方式页面明确标记：
  - 已可用：免费报名、余额支付、线下收款
  - 待服务商配置：微信支付、支付宝
  - 未完成自动化：真实退款、账单拉取、代理打款
- 未配置真实支付前，默认关闭微信/支付宝。

优先级：P0。

### 4. 系统设置页面过于复杂，重复出现短信配置

现象：

`系统设置 -> 运营设置` 中短信配置出现两段相似内容：

- “短信验证码服务”
- 又出现一次“短信验证码服务”

影响：

- 用户不知道应该填哪一组。
- 容易误配置。

建议：

- 合并短信配置区域。
- 拆分页面：
  - 运营设置
  - 支付设置
  - 短信通知
  - 页面主题
  - 部署配置

优先级：P0。

### 5. 超管“拥有所有权限”的产品表达还不够清晰

现状：

后端很多地方已区分：

- 平台超管
- 商家超管
- 运营
- 财务
- 签到

但页面上不够直观。

建议：

- 后台顶部增加“当前身份能力说明”
- 例如平台超管显示：
  - 可管理全平台商家、活动、订单、会员、公益池、系统设置
  - 余额充值/扣减仅平台超管可操作
- 商家账号显示：
  - 只能管理本商家活动、报名、订单
  - 不能调整会员余额

优先级：P0。

## 四、P1 建议尽快优化的问题

### 1. 分类体系需要升级

当前分类比较简单，只是活动分类。

建议升级为：

- 平台全局分类
- 商家自定义分类
- 分类图标
- 分类封面
- 分类是否前台展示
- 分类排序
- 分类适用场景：活动、课程、商城

建议默认分类：

- 传统文化
- 国学经典
- 书法美学
- 公益活动
- 创业成长
- 亲子教育
- 健康生活
- 城市沙龙

### 2. 新建活动流程需要做成向导式

当前活动编辑项非常多：

- 标题、封面、详情、须知
- 时间、地点、地图
- 票种、会员等级
- 报名表单
- 讲师
- 图文模块
- 群二维码
- 渠道码

建议改成 5 步：

1. 基础信息
2. 时间地点
3. 报名和票种
4. 详情装修
5. 发布和预览

这样新手商家不容易漏填。

### 3. 首页装修功能需要更“可视化”

目前 H5 全局装修功能已经有模块配置，但对普通运营不够直观。

建议：

- 左侧模块列表
- 中间手机预览
- 右侧配置面板
- 支持上传图标、选中态图标
- 每个模块提供示例说明

尤其是底部菜单，建议补齐：

- 默认图标
- 选中图标
- 菜单文案
- 跳转路径
- 是否显示
- 权限显示规则

### 4. 会员管理还缺“经营动作”

当前会员资料、余额、积分、报名、订单已经有基础能力。

建议增加：

- 批量导入会员
- 会员标签筛选
- 按消费金额筛选
- 按报名次数筛选
- 按最近活跃筛选
- 一键发送通知
- 会员详情操作日志

余额操作要继续保持：

```text
只有平台超级管理员可以充值、扣减、调整余额。
```

### 5. 文化大使招募页需要配套运营闭环

已完成：

- 独立 H5 落地页
- 后台配置
- 案例管理
- 申请列表

建议补充：

- 申请来源渠道
- 跟进人
- 下次跟进时间
- 线索等级：高意向、中意向、低意向
- 一键复制微信沟通话术
- 导出 Excel

## 五、P2 中期升级建议

### 1. 数据中心要从“看数据”升级为“给建议”

现在数据中心有总览、趋势、排行、风险。

建议增加运营建议：

- 浏览高、报名低：优化标题、封面、价格
- 报名高、支付低：检查支付方式、付款说明
- 支付高、签到低：加强开课提醒
- 退款高：检查活动交付和退款规则
- 线下收款占比高：提醒对账风险

### 2. 订单详情增加完整时间线

建议展示：

- 创建订单
- 选择支付方式
- 支付成功 / 等待线下确认
- 报名审核
- 签到
- 退款申请
- 退款完成
- 公益金计提
- 公益金保留或冲回

这样客服和财务排查会快很多。

### 3. 公益基金需要公开公示页

当前已有：

- 我的公益贡献
- 公益池
- 公益流水
- 公益项目
- 公益大使勋章

建议增加公开页：

- 平台公益池总额
- 项目进度
- 拨付记录
- 凭证图片
- 已完成公益事项
- 参与用户数

注意：文案继续用“公益金 / 公益贡献”，不要做成用户公开募捐。

### 4. 手机管理端需要做成现场工具

建议底部固定导航：

- 活动
- 报名
- 订单
- 签到
- 我的

现场人员最需要：

- 快速搜索手机号
- 扫码核销
- 手动核销
- 异常备注
- 今日活动
- 待确认线下收款

## 八、验收清单

### 分类管理验收

- 平台超管能进入分类管理。
- 平台超管能新增、编辑、停用全局分类。
- 商家管理员能新增、编辑、停用本商家分类。
- 平台活动只能选择平台分类。
- 商家活动只能选择商家分类。
- H5 分类筛选展示正确。

### 活动管理验收

- 平台超管打开“全部活动”不横向混乱。
- 待审核活动突出“审核”按钮。
- 报名中活动突出“预览、渠道、下架”。
- 更多菜单内能找到二维码、海报、复制链接、审核记录。

### 支付和订单验收

- 关闭支付方式后，H5 不展示。
- 后端直接调用关闭的支付方式会拒绝。
- 线下收款订单能被财务确认。
- 确认后报名状态、订单状态、公益金同步变化。

### 会员资产验收

- 平台超管可编辑会员资料、重置密码、调整余额。
- 商家/运营不可调整余额。
- H5 用户可修改头像、昵称、密码。
- 用户余额、公益金、报名记录展示一致。

## 九、最终建议

当前项目下一阶段不应该继续盲目加新功能，而应该先做一次“产品收口”：

```text
把已有功能的入口、权限、说明、状态、验收路径整理清楚。
```

最优先的是分类入口、活动列表按钮、系统设置拆分、支付状态说明。  
这些修完后，后台会从“功能堆叠”变成“运营人员能顺着用”的系统。

## 十、上线运营核心闭环验收补充

检查日期：2026-06-18  
检查口径：能不能收钱、能不能交付、能不能运营、出错能不能兜住。

### 1. 报名与活动订单

结论：可上线但需备注。

- 已具备真实报名记录、订单记录、线下收款、余额支付、报名审核、签到码和用户报名详情。
- 未配置真实微信/支付宝时，后端会拒绝创建真实支付，前台应走线下收款或余额支付，不应显示假支付成功。
- 仍需继续验收重复报名、库存满员候补、优惠码、积分抵扣、订单超时关闭和后台确认收款后的前台状态同步。

### 2. 课程购买与已购课程

结论：可上线但需备注。

- 已购课程已从 `user_learning` 读取，不再是纯前端假数据。
- 付费课程未确认收款前会拦截播放器；后台确认课程线下收款后才写入学习权限。
- 学习进度已支持按课时写回，并汇总到课程级进度。
- 课程在线微信/支付宝支付仍未接入正式生产链路，当前建议只开放线下确认收款和免费课程。

### 3. 支付与退款

结论：不可上线必须修，除非运营明确只使用线下收款/余额支付。

- 线下收款、余额支付、订单状态和退款申请已有基础闭环。
- 真实微信/支付宝支付、真实退款、支付验签、服务商账单和自动对账仍不能作为生产收费主链路依赖。
- 上线前必须确认后台支付方式默认关闭微信/支付宝；若要正式收钱，必须单独完成真实支付商户配置、回调验签、退款回调和对账验收。

### 4. 学员动态点赞、评论与审核

结论：可上线但需备注。

- 点赞已经有真实点赞记录，能防重复点赞。
- 评论会进入待审核状态，后台评论审核通过后才公开展示。
- 本次修复了两个验收问题：评论待审核时前台不再直接增加公开评论数；评论列表接口会按当前商家校验动态归属，避免跨商家读取评论。

### 5. 区域保护与定位分发

结论：可上线但需备注。

- 已有 `tenant_regions` 区域表和定位匹配接口，前台可在没有明确 `tenantCode` 时按定位匹配商家。
- 用户拒绝定位或当前位置无匹配时，应保留手动切换商家/城市能力。
- 仍需运营配置每个商家的经纬度、半径、排他状态和优先级；复杂街道边界、多边形围栏、定位命中日志属于后续增强。

### 6. 当前必须继续抽测的场景

- 未登录用户点击报名、课程购买、点赞、评论、打卡时必须跳登录。
- 付费活动报名后进入待付款或待确认，不允许无支付配置直接成功。
- 后台确认线下收款后，用户报名详情、我的订单、签到码状态同步变化。
- 付费课程后台未确认前不能播放，确认后出现在我的课程并可写学习进度。
- 评论提交后只提示待审核，后台通过后前台才增加评论数并展示。
- 切换商家/区域后，首页、活动、课程、订单、我的课程不能串商家。
