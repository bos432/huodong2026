## 服务器部署（2.0 上线时用）

```bash
# 登录服务器后
cd /www/wwwroot/rd.chaimen666.com

# 拉取最新代码
git fetch --all
git checkout feature/qiwai-ui-experiment
git pull origin feature/qiwai-ui-experiment

# 安装依赖
npm install --legacy-peer-deps
npm --prefix apps/api install --legacy-peer-deps
npm --prefix apps/admin install --legacy-peer-deps
npm --prefix apps/mobile install --legacy-peer-deps

# 构建
npm --prefix apps/api run build
npm --prefix apps/admin run build
npm --prefix apps/mobile run build:h5

# 需要先创建数据库表（新实体）
# 如果开启了 TypeORM synchronize 会自动创建
# 否则需要手动执行 migration

# 重启服务
/www/server/nodejs/v22.22.3/bin/pm2 restart activity-api --update-env

# 刷新 H5 静态文件
# 确保 nginx 指向 apps/mobile/dist/build/h5/
nginx -s reload
```

### 方法二：使用服务器备份

```bash

# 1. 先备份当前 2.0 代码

tar --exclude='./node_modules' --exclude='./apps/*/node_modules' --exclude='./.git' -czf runtime/rollback-backups/pre-rollback-$(date +%Y%m%d%H%M%S)/2.0-backup.tgz .

# 2. 解压 1.0 备份
cd runtime/rollback-backups/pre-ui-20260617
tar -xzf site-files.tgz -C ../../../

# 3. 恢复数据库（如有必要）
npm run db:restore  # 如果有这个命令
# 或手动导入备份文件
# gunzip < backups/mysql/reader-20260617-034515.sql.gz | mysql -u root -p reader

# 4. 重启
npm --prefix apps/api run build
npm --prefix apps/mobile run build:h5
```
