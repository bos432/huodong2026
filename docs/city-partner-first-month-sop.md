# 七维文化城市合伙人首月运营 SOP

更新时间：2026-06-12

## 1. 首月目标

首月不追求大规模扩张，目标是跑通一个城市的最小经营闭环：

> 开通后台 -> 发布 2-3 场活动 -> 完成报名收款签到 -> 打标签 -> 做复盘 -> 判断是否继续投入。

## 2. 第 1 周：开通与准备

| 事项 | 负责人 | 验收标准 |
| --- | --- | --- |
| 确认城市合伙人资料 | 平台 + 合伙人 | 城市、联系人、电话、场地、讲师方向明确 |
| 开通商家租户 | 平台 | 商家后台可登录，商家资料正确 |
| 创建员工账号 | 商家管理员 | 运营、财务、签到账号可登录 |
| 配置收款方式 | 商家财务 | 收款主体和收款账户已启用 |
| 设置运营说明 | 商家运营 | 客服、退款、发票、线下付款说明完整 |
| 确定首批活动 | 平台 + 合伙人 | 至少 2 个活动主题、时间、价格、讲师、场地明确 |

后台检查：

- 商家概览能看到平台授权状态。
- 活动发布审核、报名审核、收款配置权限清楚。
- 商家资料、运营设置、员工账号已完成。

## 3. 第 2 周：发布样板活动

| 事项 | 负责人 | 验收标准 |
| --- | --- | --- |
| 创建活动 | 商家运营 | 活动标题、封面、时间、地点、价格、报名字段完整 |
| 补充详情内容 | 商家运营 | 活动亮点、流程、适合人群、报名须知清楚 |
| 设置票种和优惠 | 商家运营 | 如需早鸟价或体验价，已配置票种/优惠码 |
| 提交平台审核 | 商家运营 | 活动进入待平台审核 |
| 平台审核 | 平台 | 合规通过后公开展示 |
| 发布推广链接 | 商家运营 | H5 活动链接可转发到社群、朋友圈、公众号 |

活动内容合规检查：

- 不出现算命、改运、破灾。
- 不承诺疗效、发财、转运。
- 不预测婚姻、疾病、财富。
- 不使用恐吓、焦虑式成交话术。

## 4. 第 3 周：报名与现场执行

| 事项 | 负责人 | 验收标准 |
| --- | --- | --- |
| 报名咨询 | 商家运营 | 咨询问题有记录，高频问题补充到活动说明 |
| 收款确认 | 商家财务 | 线下收款订单及时确认 |
| 报名名单检查 | 商家运营 | 报名人数、付款状态、特殊备注清楚 |
| 现场签到 | 商家签到 | 扫码核销，异常报名可查询 |
| 现场转化 | 商家运营 | 收集会员、训练营、复购意向 |
| 用户标签 | 商家运营 | 标记活跃用户、高意向用户、复购线索 |

建议标签：

| 标签 | 使用场景 |
| --- | --- |
| 活动活跃用户 | 已报名并到场的用户 |
| 高意向用户 | 现场咨询后续课程或会员 |
| 复购线索 | 对训练营、年卡、私董会有兴趣 |
| 讲师资源 | 可能成为内容贡献者 |
| 重点维护 | 有社群影响力或转介绍能力 |

## 5. 第 4 周：复盘与决策

| 事项 | 负责人 | 验收标准 |
| --- | --- | --- |
| 活动复盘 | 商家运营 | 查看报名、付款、签到、评价和漏斗数据 |
| 财务对账 | 商家财务 | 订单、退款、实收、异常对账清楚 |
| 用户复盘 | 商家运营 | 用户标签、复购线索和社群沉淀完成 |
| 平台复盘 | 平台 + 合伙人 | 判断下月是否加场、换主题或调整价格 |
| 下月排期 | 商家运营 | 至少确定 2 场活动计划 |

复盘问题清单：

- 哪个渠道带来最多报名？
- 哪个活动主题更容易成交？
- 报名页是否解释清楚用户收益？
- 价格是否阻碍转化？
- 到场率是否达标？
- 是否有退款或投诉？
- 现场是否产生会员或训练营线索？
- 合伙人是否具备稳定每月 2 场活动能力？

## 6. 首月验收标准

| 指标 | 达标线 | 决策 |
| --- | ---: | --- |
| 活动数 | 2 场以上 | 达标后进入月度运营 |
| 单场报名 | 10 人以上 | 低于 10 人需调整主题或渠道 |
| 已收款订单 | 至少 1 场跑通 | 未跑通不扩大投放 |
| 签到率 | 70% 以上 | 低于 70% 需优化提醒和活动吸引力 |
| 退款率 | 5% 以下 | 高于 5% 需复查宣传与交付 |
| 复购线索 | 每场至少 3 个 | 无线索需优化现场转化 |
| 下月排期 | 至少 2 场 | 无排期不建议升级合作 |

## 7. 系统验收命令

本地演示和验收可使用：

```bash
npm run seed:qiwai-demo
npm run smoke:qiwai-demo
npm run report:qiwai-demo
```

命令作用：

| 命令 | 作用 |
| --- | --- |
| `npm run seed:qiwai-demo` | 创建三城合伙人、账号、收款主体和样板活动 |
| `npm run smoke:qiwai-demo` | 验证登录、隔离、报名、收款、签到、标签、复盘、审核 |
| `npm run report:qiwai-demo` | 生成三城样板经营报告 |

## 8. 继续投入判断

建议继续投入：

- 首月能稳定组织 2 场活动。
- 有真实报名和收款。
- 到场率、退款率正常。
- 用户愿意加入社群或咨询后续课程。
- 合伙人能按时复盘并愿意调整。

建议暂停或降低投入：

- 活动迟迟无法落地。
- 用户投诉明显。
- 合伙人不配合财务对账。
- 内容存在合规风险。
- 只有招商意愿，没有执行能力。
