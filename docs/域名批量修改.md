# 域名批量修改

本文档用于慢π系统后续更换线上域名时，批量修改 H5、后台、API、跨域白名单和相关回调配置。

适用默认部署方式：

- H5：首页使用根路径，例如 `https://new.example.com/`
- 后台：使用 `/admin`，例如 `https://new.example.com/admin`
- API：同域名反向代理 `/api`，例如 `https://new.example.com/api`
- 上传文件：同域名 `/uploads`

如果后续拆分成独立域名，例如 `h5.example.com`、`admin.example.com`、`api.example.com`，请看文末“拆分域名写法”。

## 0. 后台批量修改入口

后台已提供可视化工具：

```text
系统设置 -> 部署配置 -> 域名批量修改
```

后台可以完成：

- 输入主域名后一键套用 H5、后台、API、CORS 和支付回调 URL。
- 支持“同域名部署”和“拆分域名”两种模式。
- 自动生成服务器修改命令。
- 自动生成验证命令。
- 检查当前配置是否仍包含 `example.com`、本地域名、空值或非 HTTPS。
- 尝试检测 H5、后台和 API Ready 是否可访问。

后台不会直接完成：

- DNS 解析。
- 宝塔 / Nginx 站点域名绑定。
- SSL 证书申请和续签。
- Nginx reload。
- PM2 / API 重启。

这样设计是为了避免域名填错时后台自己被改到无法访问。正确流程是：

```text
后台填写并套用 -> 保存设置 -> 复制服务器命令 -> 到服务器执行 -> 宝塔/Nginx/SSL 同步处理 -> 回后台或浏览器验证
```

## 1. 一键修改生产环境域名

在服务器执行，先把 `new.example.com` 换成你的新域名：

```bash
cd /www/wwwroot/rd.chaimen666.com

NEW_DOMAIN="new.example.com"
NEW_ORIGIN="https://${NEW_DOMAIN}"
COMMIT=$(git rev-parse --short HEAD)

cp deploy/.env.production "deploy/.env.production.bak.$(date +%F-%H%M%S)"

sed -i "s|^BUILD_COMMIT=.*|BUILD_COMMIT=${COMMIT}|" deploy/.env.production
sed -i "s|^BUILD_TIME=.*|BUILD_TIME=$(date -Iseconds)|" deploy/.env.production
sed -i "s|^CORS_ORIGIN=.*|CORS_ORIGIN=${NEW_ORIGIN}|" deploy/.env.production
sed -i "s|^PUBLIC_H5_ORIGIN=.*|PUBLIC_H5_ORIGIN=${NEW_ORIGIN}|" deploy/.env.production
sed -i "s|^PUBLIC_ADMIN_ORIGIN=.*|PUBLIC_ADMIN_ORIGIN=${NEW_ORIGIN}/admin|" deploy/.env.production
sed -i "s|^PUBLIC_API_ORIGIN=.*|PUBLIC_API_ORIGIN=${NEW_ORIGIN}|" deploy/.env.production

grep -E "BUILD_COMMIT|BUILD_TIME|CORS_ORIGIN|PUBLIC_H5_ORIGIN|PUBLIC_ADMIN_ORIGIN|PUBLIC_API_ORIGIN" deploy/.env.production
```

## 2. 重启后端 API

```bash
cd /www/wwwroot/rd.chaimen666.com

PM2=/www/server/nodejs/v22.22.3/lib/node_modules/pm2/bin/pm2

export NODE_ENV=production
export BUILD_COMMIT=$(git rev-parse --short HEAD)
export BUILD_TIME=$(date -Iseconds)
export CORS_ORIGIN="https://new.example.com"
export PUBLIC_H5_ORIGIN="https://new.example.com"
export PUBLIC_ADMIN_ORIGIN="https://new.example.com/admin"
export PUBLIC_API_ORIGIN="https://new.example.com"

$PM2 restart activity-api --update-env
$PM2 save

curl http://127.0.0.1:3000/api/health/ready
```

正常应返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "ready": true,
    "api": "up",
    "database": "up"
  }
}
```

## 3. 修改宝塔 / Nginx 站点域名

在宝塔面板或 Nginx 配置中同步处理：

- 把旧域名替换为新域名。
- 给新域名申请并绑定 SSL 证书。
- 确认 `/api/` 仍然反向代理到 `127.0.0.1:3000`。
- 确认 `/uploads/` 能正常访问上传文件。
- 确认 H5 根路径和 `/admin` 后台路径都能访问。

可用命令查找当前站点配置：

```bash
grep -R "server_name.*旧域名" -n /www/server/panel/vhost/nginx /www/server/nginx/conf 2>/dev/null
```

修改后重载 Nginx：

```bash
nginx -t
/etc/init.d/nginx reload
```

如果服务器使用宝塔封装命令，也可以在宝塔面板里点“重载配置”。

## 4. 后台系统设置同步

登录后台：

```text
https://new.example.com/admin
```

进入：

```text
系统设置 / 部署配置
```

同步填写：

```text
H5 域名：https://new.example.com
后台域名：https://new.example.com/admin
API 域名：https://new.example.com
```

保存后刷新后台和 H5。

## 5. 支付回调域名

如果真实支付还没开启，可以先不改支付回调。

如果后续开启微信/支付宝真实支付，需要同步修改 `.env.production` 和支付平台后台。

```bash
cd /www/wwwroot/rd.chaimen666.com

NEW_ORIGIN="https://new.example.com"

sed -i "s|^WECHAT_PAY_NOTIFY_URL=.*|WECHAT_PAY_NOTIFY_URL=${NEW_ORIGIN}/payment/wechat/callback|" deploy/.env.production
sed -i "s|^ALIPAY_NOTIFY_URL=.*|ALIPAY_NOTIFY_URL=${NEW_ORIGIN}/payment/alipay/callback|" deploy/.env.production
sed -i "s|^MALL_WECHAT_PAY_NOTIFY_URL=.*|MALL_WECHAT_PAY_NOTIFY_URL=${NEW_ORIGIN}/payment/mall/wechat/callback|" deploy/.env.production

grep -E "WECHAT_PAY_NOTIFY_URL|ALIPAY_NOTIFY_URL|MALL_WECHAT_PAY_NOTIFY_URL" deploy/.env.production
```

同时到对应平台后台修改：

- 微信支付商户平台：支付通知 URL。
- 支付宝开放平台：异步通知 URL。
- 如果使用微信 JSAPI/H5，还要检查授权域名、业务域名、JS 接口安全域名。

真实支付未验收前继续保持：

```env
REAL_PAYMENT_ENABLED=false
PAYMENT_SANDBOX_ENABLED=false
```

## 6. 微信 / H5 真机相关域名

如果要在微信内打开 H5 或分享海报，需要检查：

- 新域名必须是 HTTPS。
- 微信公众号 / 开放平台 / 小程序后台需要加入新域名。
- H5 域名、API 域名、上传域名需要加入合法域名或业务域名。
- 分享卡片、海报二维码、扫码回流都要用新域名复测。

真机验证入口示例：

```text
https://new.example.com/?tenantCode=qiwai-showcase#/
https://new.example.com/?tenantCode=qiwai-showcase#/pages/activity/list
https://new.example.com/?tenantCode=qiwai-showcase#/pages/community/index
```

## 7. 修改后验证

执行：

```bash
curl -I https://new.example.com
curl https://new.example.com/api/health/ready
curl "https://new.example.com/api/public/homepage?tenantCode=qiwai-showcase"
```

浏览器打开：

```text
https://new.example.com/?tenantCode=qiwai-showcase#/
https://new.example.com/admin
```

至少检查：

- H5 首页能打开。
- 活动列表能加载。
- 活动详情能打开。
- 登录或验证码入口能打开。
- 报名、我的报名、共修动态能正常请求接口。
- 后台能登录。
- 上传图片能正常显示。
- 浏览器控制台没有明显接口跨域错误。

## 8. 回滚

如果新域名配置后出现问题，可以回滚 `.env.production`：

```bash
cd /www/wwwroot/rd.chaimen666.com

ls -lt deploy/.env.production.bak.*
cp deploy/.env.production.bak.替换为具体时间 deploy/.env.production

PM2=/www/server/nodejs/v22.22.3/lib/node_modules/pm2/bin/pm2
$PM2 restart activity-api --update-env
$PM2 save
```

同时把宝塔 / Nginx 站点域名和 SSL 证书切回旧域名。

## 9. 拆分域名写法

如果以后不是一个域名，而是拆成三个域名：

```text
H5：https://h5.example.com
后台：https://admin.example.com
API：https://api.example.com
```

则执行：

```bash
cd /www/wwwroot/rd.chaimen666.com

H5_ORIGIN="https://h5.example.com"
ADMIN_ORIGIN="https://admin.example.com"
API_ORIGIN="https://api.example.com"

cp deploy/.env.production "deploy/.env.production.bak.$(date +%F-%H%M%S)"

sed -i "s|^CORS_ORIGIN=.*|CORS_ORIGIN=${H5_ORIGIN},${ADMIN_ORIGIN}|" deploy/.env.production
sed -i "s|^PUBLIC_H5_ORIGIN=.*|PUBLIC_H5_ORIGIN=${H5_ORIGIN}|" deploy/.env.production
sed -i "s|^PUBLIC_ADMIN_ORIGIN=.*|PUBLIC_ADMIN_ORIGIN=${ADMIN_ORIGIN}|" deploy/.env.production
sed -i "s|^PUBLIC_API_ORIGIN=.*|PUBLIC_API_ORIGIN=${API_ORIGIN}|" deploy/.env.production

grep -E "CORS_ORIGIN|PUBLIC_H5_ORIGIN|PUBLIC_ADMIN_ORIGIN|PUBLIC_API_ORIGIN" deploy/.env.production
```

如果真实支付开启，回调地址应使用 API 域名：

```bash
sed -i "s|^WECHAT_PAY_NOTIFY_URL=.*|WECHAT_PAY_NOTIFY_URL=${API_ORIGIN}/payment/wechat/callback|" deploy/.env.production
sed -i "s|^ALIPAY_NOTIFY_URL=.*|ALIPAY_NOTIFY_URL=${API_ORIGIN}/payment/alipay/callback|" deploy/.env.production
sed -i "s|^MALL_WECHAT_PAY_NOTIFY_URL=.*|MALL_WECHAT_PAY_NOTIFY_URL=${API_ORIGIN}/payment/mall/wechat/callback|" deploy/.env.production
```

## 10. 最简记忆版

普通同域名换域名，只记这一句：

```text
改 deploy/.env.production 里的 CORS_ORIGIN、PUBLIC_H5_ORIGIN、PUBLIC_ADMIN_ORIGIN、PUBLIC_API_ORIGIN，
再改宝塔/Nginx 站点域名和 SSL，
后台系统设置同步保存，
重启 API，
最后打开 H5 和后台验收。
```
