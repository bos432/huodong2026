## 6. 发现的问题

### P0 / 上线阻塞：短信服务未配置

- 现象：新用户通过验证码登录接口失败。
- 接口：`POST /api/public/auth/h5-code`
- 返回：`短信服务未配置，请在后台系统设置中配置短信服务`
- 影响：新用户无法使用手机号验证码注册/登录；手机号改绑、找回类链路也会受影响。
- 建议：上线前必须配置短信服务，或临时把运营入口改为“后台预创建账号 + 密码登录”并明确提示。

### P1 / 高优先级：签到核销页窄屏布局异常

- 现象：右侧浏览器当前宽度约 `385px` 时，后台签到页主内容被侧边栏挤压，表单卡片宽度约 `42px`，输入框宽度为 `0`。
- 影响：现场工作人员如果使用窄屏窗口、平板或小屏设备，无法手动输入签到码。
- 业务接口状态：`POST /admin/check-ins` 正常；问题集中在后台 UI 响应式布局。
- 建议：后台布局在窄屏下折叠侧边栏，或给主内容设置合理 `min-width` 和横向滚动；签到页表单卡片设置 `width: min(720px, 100%)` 且避免被父容器压缩到 0。

### P1 / 高优先级：店铺/代理测试账号不可用

- 现象：`showcase_store_owner`、`showcase_agent_owner`、`showcase_store_finance` 使用既有测试密码返回 `401 用户名或密码错误`。
- 影响：本轮未能完成店铺/代理/店铺财务角色的真实浏览器验收。
- 建议：重置或重新生成这些演示账号，并把账号清单写入运维手册；之后补测商城店铺、代理结算、店铺财务权限。

### P1 / 高优先级：营销弹窗缺少“未生效原因”提示

- 现象：后台列表里弹窗为停用，公开接口返回空。运营视角容易理解成“功能没用”。
- 影响：运营配置后不知道为什么前台不弹。
- 建议：营销弹窗页面增加“生效检测/预览投放”：
  - 当前是否启用。
  - 是否在投放时间内。
  - 当前页面和平台是否命中。
  - 当天频次是否已被本地缓存拦截。
  - 公开接口实时返回结果。

### P2 / 中优先级：广告计划缺少图片

- 现象：广告中心提示 `自有广告缺少图片`，公开广告位返回图片为空。
- 影响：首页广告位商业展示效果弱，也不适合正式投放。
- 建议：广告计划保存时对自有广告强制校验 HTTPS 图片；没有图片时自动使用商家默认广告图或系统兜底图。

### P2 / 中优先级：API 发布版本与本地开发 HEAD 不一致

- 现象：线上 API ready 显示 commit `9260d75`，本地当前分支 HEAD 为 `a6fffcb1`。
- 影响：如果最近仅发布了后台静态包，API 不一定包含本地最新提交；排查问题时容易混淆。
- 建议：发布后台静态包时同时记录前端静态包 commit；API、Admin、H5 分别显示构建 commit，便于定位线上版本。

### P2 / 中优先级：首页装修内容偏长且有重复运营区块

- 现象：H5 首页登录后可见多组活动、课程、商城、品牌、动态模块，运营信息较长。
- 影响：试运营可用，但用户浏览路径偏长，重点活动/报名转化会被稀释。
- 建议：上线前压缩首页首屏：保留搜索/公告/精选活动/商城广告/底栏，其他内容下沉到二级页或专题页。

## 12. 上线部署执行方案

### 12.1 服务器部署顺序

1. 备份数据库和当前静态包。
2. 拉取最新代码。
3. 安装依赖，如 lockfile 变化则执行 `npm install` 或对应部署环境依赖安装命令。
4. 构建 API：`npm --prefix apps/api run build`。

6. 重启 API 服务。
7. 构建 Admin：`npm --prefix apps/admin run build`。
8. 构建 H5：`npm --prefix apps/mobile run build:h5`。
9. 构建小程序包：`VITE_API_BASE=https://rd.chaimen666.com/api VITE_DEFAULT_TENANT_CODE=qiwai-showcase npm --prefix apps/mobile run build:mp-weixin`。
10. 发布静态包：`WEBROOT=... ADMIN_WEBROOT=... npm run publish:webroot`。

14. 执行角色验证脚本。

## 13. 最终上线判断

整改后结论：**代码侧已达到可部署到线上做最终上线验收的状态。**

当前仍不建议直接跳过线上验收宣布正式上线，原因如下：

- 真实短信发送依赖腾讯云账号、签名、模板和密钥，必须在生产环境配置后验证。
- 小程序端必须通过微信开发者工具或体验版验证登录、装修、弹窗、广告和跳转限制。
- 线上 API/Admin/H5 版本需要部署后确认一致，避免只发布静态包或只重启 API 造成版本错位。

满足以下条件后，可判定为可上线运营：

1. 生产短信测试发送成功。
2. 新用户验证码登录成功。
3. H5 报名、线下收款确认、签到核销、会员统计更新闭环通过。
4. 店铺、代理、财务、签到、多角色权限复验通过。
5. 营销弹窗和广告位在 H5 生效，小程序端构建包验证通过。
6. 后台版本卡片显示 API/Admin/H5 均为预期版本，或版本不一致原因已记录。
