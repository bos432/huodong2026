# 后台装修使用教程

## 1. 后台装修是什么

后台装修用来配置前台 H5/小程序页面的模块、顺序、文案、图片、按钮链接、底部导航和“我的”页面入口。

简单理解：

- 活动、课程、共修、商城这些业务数据来自各自的业务模块。
- 装修页负责决定这些内容在前台页面上怎么展示。
- 保存模块后，用户端刷新页面即可看到最新效果。

## 2. 入口位置

后台路径：

```text
平台端 -> 慢π运营 -> 前台全局装修
```

商家如果被平台超管授予 `homepage.manage` 权限，也可以进入自己的前台装修入口。

## 3. 平台默认装修和商家独立装修

这是最容易混淆的地方。

| 场景 | 含义 | 影响范围 |
| --- | --- | --- |
| 平台全局默认装修 | 平台超管没有选择具体商家时编辑的装修 | 没有单独装修的商家会继承这一套 |
| 商家独立装修 | 平台超管选中某个商家，或商家自己进入装修时编辑的装修 | 只影响当前商家的前台 |

推荐做法：

1. 先把平台默认装修做成一套可用基础版。
2. 对重点城市或样板商家，再做商家独立装修。
3. 如果某个商家页面改乱了，优先恢复该商家的页面，不要误改平台默认装修。

## 4. 红框工具条说明

| 按钮/控件 | 作用 | 注意事项 |
| --- | --- | --- |
| 选择页面 | 选择要装修的前台页面，例如首页、活动列表、动态详情、我的页面 | 不同页面的模块互不相同 |
| 选择商家 | 平台超管切换装修范围 | 不选商家是平台默认装修，选中商家是独立装修 |
| 发布前预览 | 打开当前页面 H5 链接并复制链接 | 适合保存前后检查实际效果 |
| 复制链接 | 只复制当前 H5 预览链接 | 可发到手机微信里测试 |
| 装修模板 | 选择一套预设模块组合 | 只是选择，不会立即生效 |
| 应用模板 | 把选中的模板应用到当前页面 | 会替换当前页面已有模块，操作前要确认 |
| 复制来源 | 选择另一个页面作为复制来源 | 配合“复制页面配置”使用 |
| 复制页面配置 | 把来源页面的模块复制到当前页面 | 会替换当前页面已有模块 |
| 跨商家复制 | 平台超管把一个商家的装修复制到另一个商家 | 会替换目标商家对应页面模块，只建议超管使用 |
| 恢复上次发布版本 | 回到本次编辑前自动保留的版本 | 适合改乱后撤回 |
| 恢复默认装修 | 重置为系统默认模块 | 会覆盖当前范围、当前页面的装修 |
| 刷新 | 重新读取后台已保存数据 | 用来确认保存后是否真正生效 |

## 5. 推荐操作流程

### 新页面从零开始

1. 选择页面。
2. 确认商家范围。
3. 选择装修模板。
4. 点击“应用模板”。
5. 修改模块文案、图片、按钮链接和排序。
6. 点击“发布前预览”检查。
7. 保存后复制链接到手机微信再检查。

### 只做小修改

1. 选择页面和商家范围。
2. 点击要改的模块。
3. 修改标题、副标题、图片或链接。
4. 先看右侧手机预览。
5. 保存模块。
6. 刷新前台页面确认。

### 页面改乱了

优先顺序：

1. 点击“恢复上次发布版本”。
2. 如果仍不可用，再考虑“复制页面配置”。
3. 最后才使用“恢复默认装修”。

### 从其他商家复制装修

这个功能只给平台超管使用，入口在工具条的“跨商家复制”。

适合场景：

- 新开城市商家，需要沿用样板商家的首页、活动页、我的页和底部导航。
- 某个商家的页面改乱了，需要用另一个商家的稳定装修覆盖。
- 平台做了一套样板商家，希望快速复制给多个商家。

操作步骤：

1. 点击“跨商家复制”。
2. 选择复制范围：
   - 当前页面：只复制一个页面，可把来源首页复制到目标首页，也可复制到目标其他页面。
   - 全部页面：逐页复制来源商家有装修模块的页面，来源为空的页面会跳过。
3. 选择来源商家。
4. 选择目标商家。
5. 当前页面模式下，再选择来源页面和目标页面。
6. 检查“复制计划”，确认没有选反。
7. 点击“确认复制”。
8. 打开目标商家的 H5 链接检查前台展示。

注意：

- 来源商家和目标商家不能相同；同一个商家内页面互相复制，用“复制页面配置”。
- 跨商家复制会替换目标商家对应页面已有模块。
- 全部页面模式不会清空来源为空的页面，避免误删目标商家的已有装修。

## 6. 模块怎么理解

| 模块 | 适合放在哪里 | 说明 |
| --- | --- | --- |
| 主视觉 | 首页顶部 | 展示品牌、活动主张、按钮和统计 |
| 公告栏 | 首页、服务中心 | 展示运营公告 |
| 快捷入口 | 首页、我的页面 | 放报名、课程、商城、签到等入口 |
| 精选活动 | 首页、活动页 | 展示推荐活动 |
| 分类标签 | 活动列表 | 帮用户筛选活动 |
| 参与者心得 | 首页、共修页 | 展示审核通过的用户心得 |
| 图片广告 | 首页、商城、课程页 | 放运营 Banner |
| 富文本 | 活动详情、报名确认 | 放说明、规则、注意事项 |
| 底部导航 | 全局 | 控制前台 5 个底部菜单 |
| 我的页 | 用户中心 | 控制个人中心头部和快捷入口 |

### 底部导航生效规则

- “前台底部导航”是全局模块，建议放在“首页”装修里统一维护。
- 菜单行右侧开关会控制单个入口是否显示；关闭后，剩余入口会自动铺满底部。
- 停用或删除整个“前台底部导航”模块后，前台不会再自动补回默认底栏。
- H5 保存后刷新页面即可看到效果；小程序端必须重新构建并导入/上传最新版后才会变化。
- 平台超管要注意商家范围：不选商家是平台默认装修，选中商家才是该商家的独立底栏。
- 小程序如果从开发工具直接打开 `pages/index/index`，默认没有 H5 URL 里的 `tenantCode`；试运营包建议构建时指定 `VITE_DEFAULT_TENANT_CODE=qiwai-showcase`，否则会读取平台默认装修。
- 如果历史上误建了多个“前台底部导航”模块，前台会按排序最后的一份作为有效配置；后台仍建议只保留一份，避免运营误判。

## 7. 上线前检查清单

- 页面选择是否正确。
- 商家范围是否正确。
- 图片是否能在 H5 和小程序里显示。
- 按钮链接是否能正常跳转。
- 底部导航是否最多 5 个入口。
- 手机微信里打开 H5 链接没有空白和错位。
- 如果是小程序，使用正确默认商家重新构建并导入最新 `apps/mobile/dist/build/mp-weixin` 后再测：

```powershell
$env:VITE_API_BASE='https://rd.chaimen666.com/api'
$env:VITE_H5_ORIGIN='https://rd.chaimen666.com'
$env:VITE_DEFAULT_TENANT_CODE='qiwai-showcase'
npm.cmd --prefix apps/mobile run build:mp-weixin
```

- 微信开发者工具里若仍显示旧页面，先点“清缓存 -> 清除全部缓存”，再重新编译。

## 8. 风险提醒

- “应用模板”“复制页面配置”“恢复默认装修”都会替换当前页面模块。
- 平台默认装修影响范围更大，操作前要确认没有选错范围。
- 小程序正式版图片需要使用合法 downloadFile 域名；推荐运营图片上传到本系统 `/uploads/`，由当前域名提供访问。
