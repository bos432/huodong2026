# 活动报名系统本地验收方案

本文档用于本地完整测试 H5、管理后台、API、MySQL 的核心业务闭环，并作为上线前试运营验收清单。

## 1. 验收目标

本轮验收目标不是简单确认页面能打开，而是确认系统可以支撑真实活动运营：

- 活动可创建、上架、编辑和下架。
- H5 可浏览活动、提交报名、查看报名状态和签到码。
- 免费活动可完成报名、审核、签到、评价闭环。
- 付费活动可完成待付款、线下收款确认、报名成功、签到闭环。
- 候补、取消、满员、截止、重复签到等异常流程有明确提示。
- 后台角色权限、导出、系统设置、上线体检可用。
- 本地构建、迁移、烟测、备份和预检流程可执行。

## 2. 验收环境准备

### 2.1 启动服务

确认 MySQL、API、后台和 H5 均已启动。

```bash
npm run dev:api
npm run dev:admin
npm run dev:mobile:h5
```

访问地址：

- H5：http://localhost:5173/
- 管理后台：http://localhost:5174/admin/login
- API 健康检查：http://localhost:3000/api/health
- API 就绪检查：http://localhost:3000/api/health/ready

默认后台账号：

```text
admin / Admin123456
```

### 2.2 自动化基线检查

先执行以下命令，确认当前代码和本地数据库处于可验收状态。

```bash
npm run build
npm --prefix apps/api run migration:show
npm run smoke
npm run smoke:flow
```

通过标准：

- `npm run build` 成功。
- `migration:show` 中所有迁移均为 `[X]`。
- `npm run smoke` 显示 `Smoke test passed.`。
- `npm run smoke:flow` 显示 `Full business flow smoke test passed.`。

## 5. 免费活动业务闭环

测试活动：`验收免费读书会 2026-06-04`

测试步骤：

1. 在 H5 打开免费活动详情。
2. 点击立即报名。
3. 填写姓名、手机号、职业/行业、备注。
4. 提交报名。
5. 进入我的活动或报名详情。
6. 确认状态为 `待审核`。
7. 后台进入报名管理。
8. 按活动筛选该报名。
9. 点击审核通过。
10. 回到 H5 报名详情并刷新。
11. 确认状态为 `报名成功`。
12. 查看签到码。
13. 后台进入签到管理。
14. 输入签到码核销。
15. H5 刷新报名详情。
16. 确认状态为 `已签到`。
17. H5 提交活动评价。
18. 后台进入评价管理查看评价。

通过标准：

- 状态流转正确：`待审核 -> 报名成功 -> 已签到`。
- 签到码可以正常生成。
- 后台核销成功。
- 重复核销提示清楚。
- 签到后可以评价。
- 后台可以查看评价。

## 6. 付费线下收款闭环

测试活动：`验收付费沙龙 2026-06-04`

测试步骤：

1. 在 H5 打开付费活动详情。
2. 点击立即报名。
3. 填写报名表并提交。
4. 查看报名详情。
5. 确认报名状态为 `待付款`。
6. 确认页面展示线下付款说明。
7. 后台进入订单管理。
8. 找到该活动对应订单。
9. 确认订单状态为 `待付款`。
10. 点击确认线下收款。
11. 填写收款备注。
12. 保存。
13. 回到 H5 报名详情并刷新。
14. 确认订单状态为 `已付款`。
15. 确认报名状态为 `报名成功`。
16. 查看签到码。
17. 后台签到核销。

通过标准：

- 付费报名不会直接成功。
- 后台确认收款后订单变为已付款。
- 后台确认收款后报名变为报名成功。
- 收款备注、收款时间、操作人有记录。
- 重复确认收款不会产生异常状态。
- 已付款报名可以签到。

## 7. 候补与满员验收

新建活动：

```text
验收候补活动 2026-06-04
```

配置：

| 配置项 | 值 |
| --- | --- |
| 状态 | 报名中 |
| 费用 | 0 |
| 需要审核 | 否 |
| 名额 | 1 |
| 允许取消 | 是 |

测试步骤：

1. H5 用户 A 报名。
2. 确认用户 A 报名成功。
3. H5 用户 B 报名。
4. 确认用户 B 进入候补。
5. 后台进入候补管理。
6. 确认可以看到用户 B 的候补记录。
7. 后台取消用户 A 的报名。
8. 后台将用户 B 补位。
9. H5 用户 B 查看报名详情。

通过标准：

- 名额为 1 时，首位用户占用正式名额。
- 满员后新用户进入候补。
- 候补记录后台可查。
- 释放名额后可以补位。
- 补位后用户状态变为报名成功。

## 8. 取消报名验收

### 8.1 允许取消

测试步骤：

1. H5 找到一个未签到报名。
2. 点击取消报名。
3. 确认取消。
4. 刷新报名详情。
5. 后台查看该报名记录。

通过标准：

- H5 状态变为 `已取消`。
- 后台状态同步为 `已取消`。
- 活动剩余名额增加。
- 已取消报名不能继续签到。

### 8.2 不允许取消

测试步骤：

1. 后台新建或编辑活动，关闭允许取消。
2. H5 报名该活动。
3. 尝试取消报名。

通过标准：

- H5 不允许取消，或取消时显示规则限制。
- 后台状态不被错误改为已取消。

## 11. 异常场景验收

必须测试以下异常：

| 场景 | 通过标准 |
| --- | --- |
| 报名截止后提交报名 | 不允许报名，提示清楚 |
| 活动下架后访问 | H5 不展示或不能报名 |

| 上传非图片文件 | 上传失败，提示清楚 |
| 上传超过 10MB 打款凭证 | 上传失败，提示清楚 |

```bash

```

```bash
npm --prefix apps/api run migration:run
```

再执行：

```bash
npm run smoke
```

- 默认 `admin / Admin123456` 已改强密码或禁用。
- Docker 和 Docker Compose 在部署机可用。
