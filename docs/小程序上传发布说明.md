# 小程序上传发布说明

本文档用于说明微信小程序从打包、预览、上传到提交审核发布的完整流程。

## 一、准备信息

正式上传前，请先确认以下信息：

| 项目 | 当前建议值 |
| --- | --- |
| H5 域名 | `https://rd.chaimen666.com` |
| API 域名 | `https://rd.chaimen666.com` |
| 后台地址 | `https://rd.chaimen666.com/admin/` |
| 小程序打包目录 | `/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/mp-weixin` |
| H5 打包目录 | `/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5` |

注意：小程序请求接口一般使用 `https://rd.chaimen666.com/api`，微信公众平台配置合法域名时只填写主域名 `https://rd.chaimen666.com`。

## 二、检查小程序 AppID

在项目中检查：

```text
apps/mobile/src/manifest.json
```

确认 `mp-weixin` 里的 `appid` 是你自己的微信小程序 AppID。

示例：

```json
{
  "mp-weixin": {
    "appid": "你的小程序AppID"
  }
}
```

如果这里不是自己的 AppID，需要修改后重新打包。

## 三、检查线上接口地址

小程序上线前，代码里不能使用：

```text
localhost
127.0.0.1
内网 IP
测试服务器地址
```

接口地址应使用线上 HTTPS 域名：

```text
https://rd.chaimen666.com/api
```

如果修改过接口地址，必须重新打包小程序。

## 四、打包微信小程序

进入项目根目录：

```bash
cd /www/wwwroot/rd.chaimen666.com
```

执行小程序打包：

```bash
npm --prefix apps/mobile run build:mp-weixin
```

打包完成后，生成目录一般是：

```text
/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/mp-weixin
```

## 五、用微信开发者工具打开项目

打开微信开发者工具，选择：

```text
导入项目
```

项目目录选择：

```text
/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/mp-weixin
```

注意：不是选择 H5 目录，也不是选择项目根目录。

## 六、配置微信公众平台服务器域名

登录微信公众平台：

```text
https://mp.weixin.qq.com
```

进入：

```text
开发管理 -> 开发设置 -> 服务器域名
```

建议配置：

```text
request 合法域名：https://rd.chaimen666.com
uploadFile 合法域名：https://rd.chaimen666.com
downloadFile 合法域名：https://rd.chaimen666.com
```

如果有 WebSocket，再配置：

```text
socket 合法域名：wss://rd.chaimen666.com
```

保存后，微信开发者工具里可以点击：

```text
详情 -> 本地设置 -> 不校验合法域名
```

本地调试时可以临时勾选；正式上传审核前，建议取消勾选并确保线上域名已配置成功。

## 七、预览测试

在微信开发者工具中点击：

```text
预览
```

用手机微信扫码测试以下内容：

1. 首页是否能打开。
2. 登录/注册是否正常。
3. 活动列表是否能加载。
4. 报名、上传图片、支付或其他关键功能是否正常。
5. 后台修改的数据，小程序端是否能同步看到。

## 八、上传版本

测试正常后，在微信开发者工具右上角点击：

```text
上传
```

建议版本号：

```text
1.0.0
```

项目备注示例：

```text
首次发布
```

如果是后续更新，可以写：

```text
修复报名流程
更新活动展示
优化首页内容
```

如果使用后台 `小程序发布` 页面上传体验版，后台会在调用微信上传前自动检查实际上传目录中的：

- `app.json`：定位权限 `scope.userLocation.desc` 超过微信 30 字限制时自动压短。
- `project.config.json`：把 `appid` 同步为后台保存的小程序 AppID，避免构建产物仍是 `touristappid`。

如果微信仍提示 `scope.userLocation exceeds 30`，优先确认后台保存的“构建目录”是否指向最新生成的 `apps/mobile/dist/build/mp-weixin`，并重新执行小程序构建。

## 九、提交审核并发布

上传完成后，回到微信公众平台：

```text
版本管理 -> 开发版本
```

点击：

```text
提交审核
```

审核通过后，再点击：

```text
发布
```

只有点击发布后，用户才能正式访问线上小程序。

## 十、常见问题

### 1. 微信开发者工具提示域名不合法

检查微信公众平台是否配置了：

```text
https://rd.chaimen666.com
```

注意必须是 HTTPS，不能是 HTTP。

### 2. 小程序里接口请求失败

优先检查：

1. 后端服务是否启动。
2. Nginx 里 `/api/` 是否代理到 `127.0.0.1:3000`。
3. 小程序代码里的接口地址是否为 `https://rd.chaimen666.com/api`。
4. 服务器 SSL 证书是否正常。

### 3. 上传图片失败

检查微信公众平台是否配置：

```text
uploadFile 合法域名：https://rd.chaimen666.com
downloadFile 合法域名：https://rd.chaimen666.com
```

同时检查 Nginx 是否有：

```nginx
location ^~ /uploads/ {
    proxy_pass http://127.0.0.1:3000/uploads/;
}
```

### 4. 改了后台设置，小程序没有变化

如果只是后台数据配置，通常刷新小程序即可。

如果改的是接口地址、AppID、构建配置、环境变量，需要重新打包并重新上传小程序。
