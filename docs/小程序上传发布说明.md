# 小程序上传发布说明

本文档用于说明微信小程序从打包、预览、上传到提交审核发布的完整流程。

## 一、准备信息

正式上传前，请先确认以下信息：

| 项目 | 当前建议值 |
| --- | --- |
| H5 域名 | `https://rd.chaimen666.com` |
| API 域名 | `https://rd.chaimen666.com` |
| 后台地址 | `https://rd.chaimen666.com/admin/` |
| 小程序打包目录 | `/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/mp-weixin` |
| H5 打包目录 | `/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/h5` |

注意：小程序请求接口一般使用 `https://rd.chaimen666.com/api`，微信公众平台配置合法域名时只填写主域名 `https://rd.chaimen666.com`。

## 二、检查小程序 AppID

在项目中检查：

```text
apps/mobile/src/manifest.json
```

确认 `mp-weixin` 里的 `appid` 是你自己的微信小程序 AppID。

示例：

```json
{
  "mp-weixin": {
    "appid": "你的小程序AppID"
  }
}
```

如果这里不是自己的 AppID，需要修改后重新打包。

## 三、检查线上接口地址

小程序上线前，代码里不能使用：

```text
localhost
127.0.0.1
内网 IP
测试服务器地址
```

接口地址应使用线上 HTTPS 域名：

```text
https://rd.chaimen666.com/api
```

如果修改过接口地址，必须重新打包小程序。

## 四、打包微信小程序

进入项目根目录：

```bash
cd /www/wwwroot/rd.chaimen666.com
```

执行小程序打包：

```bash
npm --prefix apps/mobile run build:mp-weixin
```

打包完成后，生成目录一般是：

```text
/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/mp-weixin
```

## 五、用微信开发者工具打开项目

打开微信开发者工具，选择：

```text
导入项目
```

项目目录选择：

```text
/www/wwwroot/rd.chaimen666.com/apps/mobile/dist/build/mp-weixin
```

注意：不是选择 H5 目录，也不是选择项目根目录。

## 六、配置微信公众平台服务器域名

登录微信公众平台：

```text
https://mp.weixin.qq.com
```

进入：

```text
开发管理 -> 开发设置 -> 服务器域名
```

建议配置：

```text
request 合法域名：https://rd.chaimen666.com
uploadFile 合法域名：https://rd.chaimen666.com
downloadFile 合法域名：https://rd.chaimen666.com
```

如果有 WebSocket，再配置：

```text
socket 合法域名：wss://rd.chaimen666.com
```

保存后，微信开发者工具里可以点击：

```text
详情 -> 本地设置 -> 不校验合法域名
```

本地调试时可以临时勾选；正式上传审核前，建议取消勾选并确保线上域名已配置成功。

## 七、预览测试

在微信开发者工具中点击：

```text
预览
```

用手机微信扫码测试以下内容：

1. 首页是否能打开。
2. 登录/注册是否正常。
3. 活动列表是否能加载。
4. 报名、上传图片、支付或其他关键功能是否正常。
5. 后台修改的数据，小程序端是否能同步看到。

### 本地快速调试

本地调试建议先构建微信小程序包，再用微信开发者工具打开最终产物目录：

```powershell
$env:VITE_API_BASE='https://rd.chaimen666.com/api'
npm.cmd --prefix apps/mobile run build:mp-weixin
& 'D:\Program Files (x86)\Tencent\微信web开发者工具\cli.bat' open --project (Resolve-Path 'apps/mobile/dist/build/mp-weixin').Path --lang zh
```

如果要生成本地预览二维码：

```powershell
$project = (Resolve-Path 'apps/mobile/dist/build/mp-weixin').Path
& 'D:\Program Files (x86)\Tencent\微信web开发者工具\cli.bat' preview --project $project --qr-format image --qr-output "$env:TEMP\manpi-mp-preview.png" --lang zh
```

常见结果：

- `AppID 不合法`：检查 `apps/mobile/src/manifest.json` 的 `mp-weixin.appid`，重新构建后确认 `apps/mobile/dist/build/mp-weixin/project.config.json` 不是 `touristappid`。
- `登录用户不是该小程序的开发者`：当前微信开发者工具登录账号未加入该小程序开发成员，需要在微信公众平台添加为开发者/体验者，或切换到已授权微信号。
- 模拟器控制台报 `Can't find variable: ...`：通常是 H5 浏览器 API 被打进小程序包，优先在 `apps/mobile/src` 搜索该变量并改成小程序兼容实现。

## 八、上传版本

测试正常后，在微信开发者工具右上角点击：

```text
上传
```

建议版本号：

```text
1.0.0
```

项目备注示例：

```text
首次发布
```

如果是后续更新，可以写：

```text
修复报名流程
更新活动展示
优化首页内容
```

如果使用后台 `小程序发布` 页面上传体验版，后台会在调用微信上传前自动检查实际上传目录中的：

- `app.json`：定位权限 `scope.userLocation.desc` 超过微信 30 字限制时自动压短。
- `app.json`：补齐微信“小程序登录服务”要求的 `miniApp.useAuthorizePage=true`，让官方授权 Page 能进入构建产物。
- `app.miniapp.json`：补齐 `identityServiceConfig`，默认 `authorizeMiniprogramType=1`、`miniprogramLoginPath=__default__`、`adaptWxLogin=false`。
- `app.wxss`：把微信小程序不支持的 `* { box-sizing: border-box; }` 通配选择器替换为显式组件选择器。
- `project.config.json`：把 `appid` 同步为后台保存的小程序 AppID，避免构建产物仍是 `touristappid`。

如果微信仍提示 `scope.userLocation exceeds 30`，优先确认后台保存的“构建目录”是否指向最新生成的 `apps/mobile/dist/build/mp-weixin`，并重新执行小程序构建。

## 九、提交审核并发布

上传完成后，回到微信公众平台：

```text
版本管理 -> 开发版本
```

点击：

```text
提交审核
```

审核通过后，再点击：

```text
发布
```

只有点击发布后，用户才能正式访问线上小程序。

## 十、常见问题

### 1. 微信开发者工具提示域名不合法

检查微信公众平台是否配置了：

```text
https://rd.chaimen666.com
```

注意必须是 HTTPS，不能是 HTTP。

### 2. 小程序里接口请求失败

优先检查：

1. 后端服务是否启动。
2. Nginx 里 `/api/` 是否代理到 `127.0.0.1:3000`。
3. 小程序代码里的接口地址是否为 `https://rd.chaimen666.com/api`。
4. 服务器 SSL 证书是否正常。

### 3. 微信登录提示“微信登录配置未完成”

微信登录需要后端拿到小程序 AppID 和 AppSecret 去调用 `jscode2session`。

配置来源按优先级读取：

1. 生产环境变量：`WECHAT_APP_ID`、`WECHAT_APP_SECRET`。
2. 后台“小程序发布管理”中已保存的 AppID 和 AppSecret。

如果后台已经显示 AppSecret“已保存”，但仍提示未配置，通常是服务器 API 还没拉取包含兜底逻辑的新代码，或 PM2 没有重启到最新构建。

服务器更新后可用下面命令验证：

```bash
curl -s -X POST https://rd.chaimen666.com/api/public/auth/wechat-login \
  -H 'Content-Type: application/json' \
  --data '{"code":"debug-code","appId":"wx4373059ed6b7793b"}'
```

如果配置已生效，但 `code` 是假的，返回应从“微信登录配置未完成”变为微信侧的 `invalid code` 或类似错误；真机/开发者工具真实登录会使用有效 `code`。

注意：`wx.login` 只返回临时 `code`，后端只能换取 `openid`、可能的 `unionid` 和 AppID 绑定关系；微信不会自动返回手机号、昵称或头像。

- 手机号：必须通过手机号授权组件或用户在“账号设置/手机管理”中单独绑定。
- 昵称/头像：登录页会优先调用微信原生 `wx.getUserProfile` 拉起旧版资料授权弹窗；如果当前基础库、开发者工具或微信规则不弹窗/不返回真实资料，登录页会立即打开“获取你的昵称、头像和登录权限”面板，面板内使用微信官方 `button open-type=chooseAvatar` 和 `input type=nickname` 保存资料后再登录。
- 已登录补资料：如果用户已经登录但仍显示默认 `微信用户xxxxxx` 或默认头像，可在“我的”页点击“完善微信资料”，进入“编辑资料”后补齐头像昵称并保存到后台。
- 兼容说明：你给的“多端身份管理/小程序登录服务”文档还要求 `app.miniapp.json` 和 `miniApp.useAuthorizePage`。系统已在小程序构建后自动补齐这些文件，但默认不启用 `adaptWxLogin`，因为启用后 `wx.login` 会被适配为 `wx.getMiniProgramCode`，后端必须改用多端应用的 `code2Verifyinfo`，不能再走普通小程序 `jscode2session`。
- 调试说明：微信官方“头像昵称填写”文档明确提示，开发者工具里的 `input type=nickname` 是 Web 组件模拟，部分情况下不能完整还原真机表现；最终应扫码到手机微信里验证头像选择和昵称候选。
- 后台会员列表能确认的微信身份是“来源端=微信小程序”“微信绑定=已绑定”“AppID=当前小程序 AppID”；这不等于已经获取手机号或微信昵称头像。

### 4. 上传图片失败

检查微信公众平台是否配置：

```text
uploadFile 合法域名：https://rd.chaimen666.com
downloadFile 合法域名：https://rd.chaimen666.com
```

同时检查 Nginx 是否有：

```nginx
location ^~ /uploads/ {
    proxy_pass http://127.0.0.1:3000/uploads/;
}
```

### 5. 改了后台设置，小程序没有变化

如果只是后台数据配置，通常刷新小程序即可。

如果改的是接口地址、AppID、构建配置、环境变量，需要重新打包并重新上传小程序。
